## 四宫格裂变协议（V4.1优化版）

### 协议版本：4.1

**目的**：根据V4.1优化策略标签智能分配四宫格资源，支持动态格子数（1-4格）和跨板连续性。

### V4.1优化功能

1. **动态格子数支持**：支持1-4格可变格子数
2. **策略标签扩展**：新增环境、特写、全景、中景等策略标签
3. **镜头运动集成**：支持推拉、摇移、跟拍、升降等镜头运动
4. **复杂度驱动**：根据复杂度评分自动调整格子数和策略
5. **跨板连续性**：支持跨分镜板的连续叙事

### 执行逻辑

#### Step 1: 读取诊断标签和复杂度评分
从九宫格输出的"核心策略"字段读取策略标签，从"复杂度评分"字段读取复杂度值。

#### Step 2: 执行智能裂变（V4.1优化）

| 策略标签      | 基础格子数 | 最大格子数 | 排列规则                  | 复杂度阈值 |
|---------------|-----------|-----------|---------------------------|------------|
| `[单镜]`     | 1格       | 1格       | 标准位置                  | < 2.0      |
| `[动势组]`   | 2格       | III格     | 连续格子（水平或垂直）    | 2.0-3.5    |
| `[蒙太奇组]` | 3格       | III格     | L型或连续三格             | 3.5-5.0    |
| `[长镜头]`   | 1-2格     | III格     | 关键节点选取              | ≥ III级    |
| `[环境]`     | 1-2格     | II格      | 环境展示                  | < 2.5      |
| `[特写]`     | 1格       | 1格       | 细节聚焦                  | -          |
| `[全景]`     | 1格       | 1格       | 场景建立                  | -          |
| `[中景]`     | 1-2格     | II格      | 角色与环境关系            | -          |

#### Step 3: 镜头运动策略分配

| 镜头运动      | 适用策略标签          | 描述                     |
|---------------|---------------------|--------------------------|
| `[推拉]`      | `[特写]`, `[单镜]`   | 镜头推进或拉远           |
| `[摇移]`      | `[全景]`, `[环境]`   | 镜头水平或垂直移动       |
| `[跟拍]`      | `[动势组]`, `[长镜头]` | 镜头跟随角色移动         |
| `[升降]`      | `[环境]`, `[全景]`   | 镜头垂直升降             |

#### Step 4: 复杂度调整算法
```
最终格子数 = 基础格子数 + 复杂度调整值

其中：
- 复杂度调整值 = 
  - 复杂度 < 2.0: 0
  - 2.0 ≤ 复杂度 < 3.5: 1
  - 复杂度 ≥ 3.5: 2

限制：最终格子数 ≤ 最大格子数
```

#### Step 5: 长节拍自动拆分规则
当满足以下条件时，自动拆分为2个分镜板：
- 复杂度 ≥ 3.5
- 关键点数量 ≥ 4
- 最终格子数需求 > 4

拆分逻辑：
1. 第一个板：Setup + Action（2-3格）
2. 第二个板：Resolution + 余波（1-2格）
3. 添加跨板连续性标记：`[跨板→]`

**示例**：
Beat 5: [动势组: 冲向门口] → 占用格1和格2
Beat 6: [单镜: 对话] → 占用格3
Beat 7: [蒙太奇组: 闪回] → 创建新板，占用新板的格1、格2、格3

#### Step 3: 插值补全
当裂变后格子不够时：
1. 严禁强行压缩：不要将多格内容挤入单格
2. 自动创建新板：
   - 当前板格子用完时，自动创建 Sequence Board 02
   - 保持节拍顺序和叙事连贯性
   - 在新板开头注明"接上板"
3. 跨板连贯性检查：
   - 角色形象、环境、光影必须一致
   - 情绪曲线必须平滑过渡

#### Step 6: 生成镜头描述（V4.1优化）
对于每个占用的格子，根据策略标签生成对应的镜头描述：

**单镜 [单镜]（1格）**：
- 画面：完整的静态场景，聚焦关键元素
- 镜头：固定镜头或轻微 `[推拉]`
- 时长：2-4秒
- 适用：对话、静态情感、特写

**动势组 [动势组]（2格）**：
- 格A：起幅画面 + 动作开始 + `[摇移]` 开始
- 格B：落幅画面 + 动作完成 + `[摇移]` 结束
- 运动连续性：注明动作轨迹和速度变化
- 时长：每格1-2秒，总时长2-4秒

**蒙太奇组 [蒙太奇组]（3格）**：
- 格A：Setup（起势） + 环境建立 + `[全景]` 或 `[环境]`
- 格B：Action（动作） + 核心转折 + `[特写]` 或 `[中景]`
- 格C：Resolution（结果） + 情绪余波 + `[推拉]` 收尾
- 三段式逻辑：建立→转折→解决
- 时长：每格1-2秒，总时长3-6秒

**长镜头 [长镜头]（1-3格）**：
- 关键节点：选取3个关键姿势
- 镜头运动：`[跟拍]` + `[升降]` 复合运动
- 连续性：注明完整动作轨迹
- 时长：每格1-2秒，总时长3-6秒

**环境展示 [环境]（1-2格）**：
- 格A：空间骨架构建 + `[全景]`
- 格B：材质皮肤覆盖 + `[摇移]` 展示细节
- 氛围：光影氛围注入描述
- 时长：每格2-3秒，总时长4-6秒

**特写镜头 [特写]（1格）**：
- 聚焦：表情、物品、细节
- 镜头：`[推拉]` 推进
- 情感：情感表达强化
- 时长：2-3秒

**全景镜头 [全景]（1格）**：
- 展示：完整场景、空间关系
- 镜头：固定或缓慢 `[摇移]`
- 功能：场景建立、空间定位
- 时长：3-4秒

**中景镜头 [中景]（1-2格）**：
- 关系：角色与环境互动
- 镜头：`[摇移]` 或固定
- 适用：对话、动作展示
- 时长：每格2-3秒

**长镜头（1-2格）**：
- 选取关键姿势，生成关键节点描述
- 标注镜头运动轨迹

#### 继承机制（V4.1优化）
- **数据来源**：从 `beat-breakdown-{集数}.md` 继承以下字段：
  1. 核心策略（含策略标签）
  2. 复杂度评分
  3. 建议格子数
  4. 关键信息点
  5. 拆解建议
  6. 最终权重
  7. 关键帧等级

- **V4.1处理方式**：
  1. 读取节拍拆解表的所有V4.1优化字段
  2. 根据复杂度评分和策略标签应用智能裂变规则
  3. 根据最终权重确定镜头运动强度
  4. 根据关键帧等级分配资源优先级
  5. 记录完整继承关系："基于节拍 {编号} [核心策略] (复杂度:{评分}) 进行裂变"

- **V4.1新增继承字段**：
  - `strategy_tags`: 策略标签列表（来自配置）
  - `complexity_threshold`: 复杂度阈值配置
  - `type_coefficients`: 类型系数配置
  - `optimization_config`: 优化配置开关
