### 2.1 4.1 策略执行逻辑补全

**目标**：补全缺失的[正反打]和[感官锚点]策略执行逻辑

**实施内容**：
- 修改 `dynamic-breakdown-engine.md`
- 在"基础框架（基于核心策略）"部分后新增"策略映射"部分
- 新增内容：
  1. **[正反打]策略（2格逻辑）**：
     - 格1: A角特写 + 台词/表情
     - 格2: B角反应镜头 + 环境资产继承
     - 轴线规则: 确保视线匹配
     - 示例：格1（A角特写）：鲁达面部特写，眼神犀利，说出威胁台词
  2. **[感官锚点]策略（1格插入规则）**：
     - 插入规则: 在节拍的情感/动作顶点后插入
     - 内容: 特写镜头 + 通感视觉化方案
     - 示例："玻璃碎裂的特写，裂痕中映射出角色的倒影"
  3. **策略优先级调整**：
     - 当[感官锚点]与其他策略冲突时，优先保证锚点的插入
     - [感官锚点]优先级：高
     - 其他策略优先级：中
     - 冲突处理流程：检测到[感官锚点] → 检查是否有其他策略冲突 → 优先执行[感官锚点]插入规则 → 调整其他策略的格子数

**测试验证**：
- [✅] [正反打]策略的2格逻辑完整定义

- [✅] 轴线规则明确
- [✅] 策略优先级调整规则清晰

### 2.3 4.3 裂变生成算法

**目标**：实现基于资产的父子裂变算法

**实施内容**：
- 修改 `dynamic-breakdown-engine.md`
- 在"V4.1字段处理规则"部分新增"父子裂变算法"
- 新增内容：
  1. **算法目标**：基于九宫格父画面的环境资产包，生成四宫格子镜头卡片
  2. **输入数据**：
     - 九宫格父画面（含环境资产包）
     - 节拍拆解的[关键信息点]
     - 策略标签类型
  3. **处理逻辑**：
     - **单镜裂变**：
       - 保持1格，但镜头角度/景深可以微调
       - 继承父画面的完整环境资产包
       - 画面描述：基于父画面的五维扩写
       - 输出：1个子镜头卡片
     - **动势组裂变**：
       - 格1: 继承父画面的起势状态 + 环境资产
       - 格2: 基于[关键信息点]生成落幅画面
       - 运动连续性：两格间必须描述运动连续性
       - 输出：2个子镜头卡片
     - **蒙太奇组裂变**：
       - 格1（Setup）：从环境资产包提取氛围，建立场景
       - 格2（Action）：父画面的高潮帧，包含核心转折
       - 格3（Resolution）：情绪回落，环境资产微调（如光影变暗）
       - 情绪曲线：Setup→Action→Resolution的情绪递进
       - 输出：3个子镜头卡片
     - **长镜头裂变**：
       - 选取1-3个关键节点（根据复杂度和关键点数量）
       - 每个节点继承环境资产但更新角色位置
       - 镜头运动：标注完整的镜头运动轨迹
       - 连续性：节点间的角色运动连续性必须描述
       - 输出：1-3个子镜头卡片
  4. **输出格式**：
     - 裂变后的子镜头卡片列表，每个卡片包含：
       - 镜头序号（1-N）
       - 继承关系（来自父格的信息）
       - 环境构造（继承+调整）
       - 摄影参数（镜头、运动、角度）
       - 画面叙事（基于裂变算法生成的画面描述）
       - 运动连续性（对于多格裂变）
       - 情绪曲线（对于蒙太奇组）
  5. **完整Python实现**：
     - `single_shot_fission()`: 单镜裂变算法
     - `action_fission()`: 动势组裂变算法
     - `montage_fission()`: 蒙太奇组裂变算法
     - `long_take_fission()`: 长镜头裂变算法
  6. **算法流程图**：
     ```
     输入：九宫格父画面 + 节拍拆解的[关键信息点]
         ↓
     识别策略标签类型
         ↓
     ├─→ [单镜] → 单镜裂变
     ├─→ [动势组] → 动势组裂变
     ├─→ [蒙太奇组] → 蒙太奇组裂变
     └─→ [长镜头] → 长镜头裂变
         ↓
     生成子镜头卡片列表
         ↓
     注入环境资产继承关系
         ↓
     输出：裂变结果（含完整继承信息）
     ```

**测试验证**：
- [✅] 单镜裂变逻辑完整
- [✅] 动势组裂变逻辑完整
- [✅] 蒙太奇组裂变逻辑完整
- [✅] 长镜头裂变逻辑完整
- [✅] 环境继承规则覆盖所有裂变类型
- [✅] 运动连续性描述规则明确
- [✅] 输出格式规范
- [✅] 算法流程图清晰

### 2.4 4.4 卡片式布局重构

**目标**：重构四宫格布局为卡片式布局

**实施内容**：
- 修改 `templates/sequence-board-template.md`
- 将"四宫格提示词"部分的表格布局重构为卡片式布局
- 新增内容：
  1. **组头共享区（Group Header）**：
     ```markdown
     ### 第{组号}组：{叙事功能}组头共享区 (Group Header)
     
     **风格**: {主导风格}
     **情绪弧线**: {从...到...}
     **环境基调**: {继承自九宫格的环境资产摘要}
     ```
  2. **镜头卡片结构**：
     ```markdown
     **镜头卡片 1** (节拍{编号})
     ├── **环境构造 (继承)**: {空间骨架摘要} | {光影系统}
     ├── **摄影参数**: {镜头} | {运动} | {角度}
     └── **画面叙事**: {基于裂变算法生成的画面描述}
     ---
     
     **镜头卡片 2** (节拍{编号})
     ├── **环境构造 (继承)**: ...
     ├── **摄影参数**: ...
     └── **画面叙事**: ...
     ```
  3. **卡片要求**：
     - 每个镜头卡片是独立的Markdown区块，便于解析
     - 使用水平线(`---`)分隔不同的镜头卡片
     - 使用树状结构(`├──`、`└──`)组织卡片内部信息
     - 环境构造必须标注继承关系
     - 摄影参数包含镜头、运动、角度三个维度
     - 画面叙事基于裂变算法生成
