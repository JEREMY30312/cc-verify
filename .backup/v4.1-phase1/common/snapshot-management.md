# 快照管理规则

> **【详细规范文档】**
>
> 本文件作为 AGENTS.md 的扩展说明，详细描述快照管理机制。
> 请参考 AGENTS.md 了解整体架构和流程。

 ---

## 核心原则

快照是项目在某一时点的完整状态备份，支持回滚到任意历史版本。

## 1. 快照类型

| 快照类型 | 创建时机 | 保留策略 | 说明 |
|----------|----------|----------|------|
| **完整快照** | CP1、CP5 | 永久保留 | 包含所有项目文件 |
| **分板快照** | CP2 | 可清理 | 包含九宫格相关文件 |
| **分组快照** | CP3 | 可清理 | 包含四宫格相关文件 |
| **参数快照** | CP4 | 可清理 | 包含动态提示词参数 |
| **手动快照** | 用户手动触发 | 用户决定 | 用户主动创建 |

### 快照内容

**完整快照包含：**
```
├── .agent-state.json
├── outputs/* (所有产出)
└── script/* (所有脚本)
```

**可选包含：**
```
├── .strategic-snapshots/snapshot.json
└── configs/* (配置文件夹)
```

## 2. 快照创建时机

### 自动创建

| 触发条件 | 快照类型 | 说明 |
|----------|----------|------|
| 用户确认前 | 完整快照 | CP1、CP5确认前自动创建 |
| 用户修改前 | 对应快照 | 修改当前阶段前创建 |
| 用户重生成前 | 对应快照 | 重生成当前阶段前创建 |
| 阶段完成 | 对应快照 | 每个确认点完成后创建 |

### 手动创建

```bash
/snapshot create "备注信息"
```

## 3. 快照保留策略

| 分支类型 | 保留策略 | 清理时机 |
|----------|----------|----------|
| 主分支 | 永久保留 | - |
| 功能分支 | 项目完成后清理 | 项目结束时 |
| 手动快照 | 用户决定 | 用户手动触发 |

## 4. 快照操作

### 查看快照列表

```bash
/snapshot list
```

输出示例：
```
可用快照：
├─ ✅ v1.0 [完整快照] 2024-01-20 14:30 - 用户确认初始节拍拆解
├─ ✅ v1.1 [完整快照] 2024-01-20 15:00 - 用户确认九宫格
├─ ✅ v1.2 [完整快照] 2024-01-20 15:30 - 用户确认四宫格
├─ ✅ v1.3 [完整快照] 2024-01-20 16:00 - 用户确认动态提示词
└─ ✅ v1.4 [完整快照] 2024-01-20 16:30 - 用户确认最终交付

分板快照（可清理）：
├─ ✅ board-v1 [分板快照] 2024-01-20 14:45
└─ ✅ board-v2 [分板快照] 2024-01-20 15:15

手动快照：
└─ ✅ manual-v1 [手动快照] 2024-01-20 14:00
```

### 预览快照

```bash
/snapshot preview <snapshot_id>
```

### 回滚到快照

```bash
/snapshot rollback <snapshot_id>
```

回滚后系统自动重新检查连贯性。

### 删除快照

```bash
/snapshot delete <snapshot_id>
```

（仅手动快照可删除）

### 对比快照

```bash
/snapshot diff <snapshot_id_1> <snapshot_id_2>
```

## 5. 快照管理界面

```
───────────────────────────────
  快照管理 - [项目名称]
───────────────────────────────

主分支快照（永久保留）：
├─ ✅ v1.0 [完整快照] 2024-01-20 14:30
├─ ✅ v1.1 [完整快照] 2024-01-20 15:00
└─ ...

分板快照（可清理）：
├─ ✅ board-v1 [分板快照] 2024-01-20 14:45
└─ ...

手动快照：
└─ ✅ manual-v1 [手动快照] 2024-01-20 14:00

───────────────────────────────
操作：
├─ [预览] - 预览快照内容
├─ [回滚] - 回滚到该版本
├─ [对比] - 与当前版本对比
├─ [删除] - 删除该快照（仅手动快照）
└─ [创建] - 创建新手动快照
```

## 6. 与用户确认流程的关系

每个确认点会自动创建对应类型的快照：

| 确认点 | 快照类型 | 快照内容 |
|--------|----------|----------|
| CP1 | 完整快照 | 所有文件 |
| CP2 | 分板快照 | 九宫格相关 |
| CP3 | 分组快照 | 四宫格相关 |
| CP4 | 参数快照 | 动态参数 |
| CP5 | 完整快照 | 所有文件 |

回滚操作：
1. 用户选择[回滚]
2. 显示可用快照列表
3. 用户选择目标快照
4. 系统回滚到该版本
5. 自动重新检查连贯性
6. 记录到操作日志
