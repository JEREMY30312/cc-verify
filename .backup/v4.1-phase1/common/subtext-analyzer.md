# 潜台词分析器规范

## 概述

本文件定义了 ANINEO V3.0 系统的潜台词分析功能，用于从表面动作中挖掘深层心理动机，并转化为可拍摄的视觉对应物。

## 分析流程

### 三层分析架构

```
输入: 场景描述文本
    ↓
[探测层] - 识别表面动作和关键词
    ↓
[翻译层] - 映射到心理动机和潜台词
    ↓  
[表现层] - 生成视觉对应物和拍摄建议
    ↓
输出: 潜台词分析结果 (三字段)
```

### 1. 探测层 (Detection Layer)

**功能**: 识别场景描述中的表面动作和关键元素

**分析规则**:
1. **动作提取**: 识别主要动词和动作描述
2. **情绪关键词**: 检测情绪相关词汇
3. **感官线索**: 识别感官描述（看、听、闻、触）
4. **对话标记**: 识别对话内容和说话方式

**技术实现**:
- 使用正则表达式匹配常见动作模式
- 关键词库匹配情绪和感官词汇
- 上下文分析确定动作意图

### 2. 翻译层 (Translation Layer)

**功能**: 将表面动作翻译为心理动机和潜台词

**映射规则表**:

| 表面动作模式 | 可能潜台词 | 心理动词 | 置信度 |
|--------------|------------|----------|--------|
| 瞪视/凝视对方 | 威慑/试探/观察 | [威慑, 试探, 观察] | 高 |
| 转身/背对离开 | 失望/愤怒/拒绝 | [失望, 愤怒, 拒绝] | 中 |
| 冷笑/讥笑 | 不屑/挑衅/嘲讽 | [不屑, 挑衅, 嘲讽] | 高 |
| 沉默/停顿 | 思考/抗拒/犹豫 | [思考, 抗拒, 犹豫] | 中 |
| 快速靠近 | 威胁/亲近/急切 | [威胁, 亲近, 急切] | 高 |
| 后退/躲避 | 恐惧/厌恶/谨慎 | [恐惧, 厌恶, 谨慎] | 高 |
| 颤抖/哆嗦 | 恐惧/激动/寒冷 | [恐惧, 激动, 寒冷] | 高 |
| 微笑/点头 | 同意/满意/掩饰 | [同意, 满意, 掩饰] | 中 |
| 握拳/咬牙 | 愤怒/决心/忍耐 | [愤怒, 决心, 忍耐] | 高 |
| 叹气/摇头 | 失望/无奈/放弃 | [失望, 无奈, 放弃] | 中 |

**分析逻辑**:
1. 匹配表面动作到最可能的潜台词（基于上下文）
2. 生成1-3个最相关的心理动词
3. 计算置信度评分（高/中/低）
4. 考虑角色关系和场景上下文

### 3. 表现层 (Expression Layer)

**功能**: 将心理动机转化为可拍摄的视觉对应物

**视觉对应物映射表**:

| 心理状态 | 视觉对应物 | 拍摄建议 | 细节程度 |
|----------|------------|----------|----------|
| 威慑 | 眯起的眼睛、紧握的拳头、挺直的脊背 | 特写眼睛、低角度拍摄 | 高 |
| 试探 | 微微前倾的身体、游移的眼神、轻敲的手指 | 中景、跟焦眼神 | 中 |
| 愤怒 | 涨红的脸颊、暴起的青筋、急促的呼吸 | 特写面部、手持晃动 | 高 |
| 恐惧 | 颤抖的嘴唇、收缩的瞳孔、后退的脚步 | 特写嘴唇、慢动作后退 | 高 |
| 思考 | 低垂的眼帘、抚摸下巴、踱步 | 中景、静态构图 | 中 |
| 掩饰 | 假笑、回避眼神、多余的小动作 | 特写嘴角、眼神跟踪 | 高 |
| 喜悦 | 发亮的眼睛、舒展的眉头、轻快的动作 | 柔光、升格拍摄 | 中 |
| 悲伤 | 低垂的肩膀、无神的眼睛、缓慢的动作 | 低调光、静态长镜头 | 高 |

**生成规则**:
1. 选择2-3个最典型的视觉对应物
2. 提供具体的拍摄建议
3. 标注细节程度（指导拍摄优先级）
4. 确保视觉对应物具体可拍摄

## 心理动词库

### 核心心理动词分类

#### A. 情绪表达类
- **积极**: 喜悦、兴奋、满足、自豪、爱慕、感激
- **消极**: 愤怒、恐惧、悲伤、厌恶、羞愧、嫉妒
- **中性**: 好奇、困惑、犹豫、期待、回忆、决定

#### B. 意图动机类
- **主动**: 追求、攻击、保护、控制、说服、展示
- **被动**: 躲避、防御、服从、等待、观察、接受
- **社交**: 连接、排斥、竞争、合作、领导、跟随

#### C. 认知过程类
- **思考**: 分析、评估、计划、回忆、理解、怀疑
- **感知**: 注意、发现、识别、感受、体验、意识

### 动词选择规则
1. **相关性优先**: 选择与表面动作最相关的动词
2. **具体性优先**: 选择具体的动词而非抽象概括
3. **可拍摄性**: 优先选择有明确视觉表现的动词
4. **数量限制**: 每个分析输出1-3个心理动词

## 输出字段定义

### 集成到节拍拆解表的新增字段

在节拍拆解表中新增以下三个字段：

```markdown
### 节拍 [编号]: [节拍名称]

**场景描述**: [原始描述]

**潜台词分析**:
- **潜台词**: [分析结果，1-2句话]
- **心理动词**: [[动词1], [动词2], [动词3]]
- **视觉对应物**: [视觉元素1]/[视觉元素2]/[视觉元素3]
- **拍摄建议**: [具体拍摄建议]
- **置信度**: [高/中/低]
```

### 字段详细说明

| 字段 | 格式 | 要求 | 示例 |
|------|------|------|------|
| **潜台词** | 字符串，1-2句话 | 揭示深层动机，自然语言表达 | "表面刁难，实则测试对方底线" |
| **心理动词** | 数组，1-3个动词 | 使用心理动词库中的标准动词 | `[挑衅, 测试, 威慑]` |
| **视觉对应物** | 字符串，2-3个元素 | 具体可拍摄的视觉元素，用"/"分隔 | "挑衅的眼神/敲击桌面的手指" |
| **拍摄建议** | 字符串，1-2句话 | 具体的拍摄技术建议 | "特写眼睛，低角度拍摄增强威慑感" |
| **置信度** | 枚举: 高/中/低 | 基于分析确定性的评分 | "高" |

## 调用时机与集成

### 集成点

**节拍分析器 (`beat-analyzer.md`)**:
- 在节拍分析完成后调用潜台词分析
- 对每个节拍的"场景描述"字段执行分析
- 将分析结果追加到节拍拆解表

**质量检查 (`quality-check.md`)**:
- 验证潜台词分析结果的完整性
- 检查心理动词的合理性
- 验证视觉对应物的可拍摄性

**导演审核**:
- 导演可以审核和修正潜台词分析
- 提供更准确的心理学解读
- 调整视觉对应物的选择

### 性能约束

**分析时间限制**:
- 每个节拍分析时间: < 500ms
- 批量分析优化: 支持并行处理
- 缓存机制: 相似场景复用分析结果

**资源使用**:
- 内存使用: < 50MB/分析
- CPU使用: 单核处理
- 可扩展性: 支持分布式处理

## 示例分析

### 示例1: 挑衅场景

**输入场景描述**:
"鲁达挑衅要求再切十斤精肉臊子"

**分析过程**:
1. **探测层**: 识别"挑衅"、"要求"动作
2. **翻译层**: 映射到"测试对方底线"潜台词
3. **表现层**: 生成视觉对应物"挑衅的眼神/敲击桌面的手指"

**输出结果**:
```
**潜台词分析**:
- **潜台词**: "表面刁难，实则测试对方底线和耐心"
- **心理动词**: `[挑衅, 测试, 威慑]`
- **视觉对应物**: "挑衅的眼神/敲击桌面的手指/微微前倾的身体"
- **拍摄建议**: "特写眼睛和手指动作，低角度增强压迫感"
- **置信度**: "高"
```

### 示例2: 恐惧场景

**输入场景描述**:
"镇关西看到鲁达眼中的杀气，不由自主后退两步"

**分析过程**:
1. **探测层**: 识别"看到"、"后退"动作，"杀气"情绪词
2. **翻译层**: 映射到"感受到生命威胁"潜台词  
3. **表现层**: 生成视觉对应物"收缩的瞳孔/颤抖的嘴唇/后退的脚步"

**输出结果**:
```
**潜台词分析**:
- **潜台词**: "感受到对方的致命威胁，本能逃避"
- **心理动词**: `[恐惧, 警觉, 逃避]`
- **视觉对应物**: "收缩的瞳孔/颤抖的嘴唇/后退时拖沓的脚步"
- **拍摄建议**: "特写瞳孔收缩，慢动作后退，手持摄影增强紧张感"
- **置信度**: "高"
```

### 示例3: 掩饰场景

**输入场景描述**:
"郑屠勉强挤出一丝笑容，说'客官说笑了'"

**分析过程**:
1. **探测层**: 识别"挤笑容"、"说"动作，"勉强"修饰词
2. **翻译层**: 映射到"用笑容掩饰不安和愤怒"潜台词
3. **表现层**: 生成视觉对应物"僵硬的嘴角/回避的眼神/紧握的拳头"

**输出结果**:
```
**潜台词分析**:
- **潜台词**: "用职业笑容掩饰内心的不安和逐渐积累的愤怒"
- **心理动词**: `[掩饰, 不安, 愤怒]`
- **视觉对应物**: "僵硬的嘴角/回避的眼神/柜台下紧握的拳头"
- **拍摄建议**: "特写嘴角和眼神的 mismatch，揭示掩饰痕迹"
- **置信度**: "中"
```

## 配置与定制

### 配置文件

在 `.agent-state.json` 中新增配置:

```json
{
  "projectConfig": {
    "enable_features": ["subtext_analysis"],
    "subtext_config": {
      "analysis_depth": "standard",  // standard/deep/minimal
      "visual_detail": "high",       // high/medium/low
      "confidence_threshold": 0.6,   // 置信度阈值
      "custom_mappings": {}          // 自定义映射规则
    }
  }
}
```

### 分析深度级别

| 级别 | 分析内容 | 处理时间 | 适用场景 |
|------|----------|----------|----------|
| **minimal** | 基础潜台词+1个心理动词 | < 200ms | 快速处理、大量节拍 |
| **standard** | 完整分析 (默认) | < 500ms | 标准影视制作 |
| **deep** | 深度心理学分析+多角度解读 | < 1000ms | 文艺片、心理剧 |

### 自定义映射

用户可以通过 `custom_mappings` 字段添加特定项目的映射规则:

```json
"custom_mappings": {
  "project_specific_actions": {
    "转佛珠": {
      "subtext": "内心焦虑但表面平静",
      "psych_verbs": ["焦虑", "平静", "祈祷"],
      "visual_cues": "快速转动的佛珠/微闭的眼睛/均匀的呼吸"
    }
  }
}
```

## 错误处理与降级

### 分析失败处理

**常见失败场景**:
1. **无法识别动作**: 场景描述过于抽象或模糊
2. **矛盾上下文**: 动作与上下文明显矛盾
3. **文化特定动作**: 不熟悉的文化特定行为

**降级策略**:
1. **简化分析**: 只输出最基础的潜台词
2. **标记低置信度**: 明确标注分析不确定性
3. **跳过分析**: 极端情况下跳过该节拍分析

### 置信度管理

| 置信度 | 标准 | 处理方式 |
|--------|------|----------|
| **高** (≥0.8) | 动作明确，上下文清晰，映射准确 | 正常输出，推荐采用 |
| **中** (0.5-0.8) | 动作较模糊，有多个可能解读 | 输出并标注"可能"，提供备选 |
| **低** (<0.5) | 动作不明确，上下文矛盾 | 标记"低置信度"，建议人工审核 |
