# 关键帧选择模块（V4.1-CRITICAL单帧原则）

## 模块用途

从节拍拆解表的完整描述中提取**戏剧张力最强的单帧画面**，供九宫格使用。

## V4.1-CRITICAL单帧原则

### 核心规则

```
⚠️ 九宫格核心原则：每格=一个关键帧 [V4.1-CRITICAL]
```

### 禁止内容

| 禁止类型 | 示例 | 原因 |
|---------|------|------|
| 连续动作序列 | "包子猪被拉走，重重按在案板" | 包含多个动作，应由四宫格拆分 |
| 时间序列词汇 | "然后"、"接着"、"随后" | 表明动作进展，违反单帧原则 |
| 蒙太奇过程 | "惊恐→系餐巾→期待" | 蒙太奇属于四宫格工作 |
| 一格多画面 | "正反打A" + "正反打B" | 一格只应一个画面 |

### 强制内容

| 必须内容 | 示例 |
|---------|------|
| 聚焦单帧 | 静态姿态 + 表情 + 眼神 |
| 格式 | [动作主体] + [静态姿态] + [表情/眼神] |
| **[关键帧]** ✅标签 | 每格必须标注 |
| **风格标签** | 简洁关键词列表 |

## 单帧提取算法

### 输入

```
节拍拆解表完整描述（scene_breakdown shot 或 beat-breakdown 整段）
└─ 从 scene-breakdown.json 的 shots[].scene_description
   或 beat-breakdown.md 的完整节拍描述
```

### 提取步骤

#### 步骤1：识别动作序列

**检测方法**：扫描完整描述中的违禁词和动作动词

```python
违禁词 = [
  "然后", "接着", "随后", "接下来", "同时", "与此同时",
  "第一", "第二", "首先", "最后",
  "一边...一边...", "一方面...另一方面...",
  "后", "之后", "接着便", "随后便",
  "→", "→", "→", "→"
]

动作动词 = [
  "拉", "冲", "滑", "扑向", "跑向", "飞向", "跳向",
  "切", "挥", "举", "砸", "拍", "踢", "推",
  "转", "闪", "避", "躲"
]
```

**示例检测**：
- "包子猪被**拉走**，**重重按在**案板" → 识别为连续动作
- "鲁达**冲出**，**滑铲接球**" → 识别为连续动作

---

#### 步骤2：定位戏剧张力最强帧

**方法**：参考 beat-breakdown.md 的"戏剧张力最强帧"列

| 场景 | 从哪里提取 |
|------|----------|
| 节拍拆解表（beat-breakdown.md）| "戏剧张力最强帧描述"列 |
| 生成数据（scene-breakdown.json）| 从完整描述提取最强片段 |

**定位规则**：

1. **如果有明确标注**：
   - 直接使用 beat-breakdown 的"戏剧张力最强帧"列
   - 示例："包子猪被按在案板，表情惊讶"

2. **如果没有明确标注**：
   - 识别完整描述中的动作节点
   - 选择戏剧张力最高的节点
   - 判断标准：
     - 关键转折点（咬腕、变身、爆炸）
     - 情绪极值（期待眼神、惊恐、愤怒）
     - 视觉高光（诞生、变身、悬浮）

---

#### 步骤3：单帧画面重新表述（核心规则）

**规则1：去除连续动作**

| 错误示例（连续动作）| 正确示例（单帧）| 去除内容 |
|-----------------|------------|---------|
| "包子猪被拉走，重重按在案板" | "包子猪被按在案板中央" | "被拉走" |
| "鲁达冲出→滑铲→接球" | "鲁达滑铲姿势清晰" | "冲出"、"接球" |
| "暴怒→周身爆气→包子悬浮" | "暗物质包子诞生：悬浮红色闪电" | "暴怒"、"周身爆气" |
| "惊恐→系餐巾→期待敲盘" | "包子猪系好餐巾后，期待敲盘" | "惊恐"、"系餐巾" |

**规则2：禁用违禁词**

| 禁用词 | 正确替换 |
|--------|---------|
| "然后" + 动作 | 去除或改为"后"（表示完成非时间序列）|
| "接着" + 动作 | 去除 |
| "随后" + 动作 | 去除 |

**规则3：聚焦单帧画面**

**强制格式**：
```
单帧画面 = [动作主体] + [静态姿态] + [表情/眼神]
```

**示例**：

| 节拍 | 单帧画面表述 |
|------|------------|
| 节拍2 | 鲁达睁开挑剔的双眼，傲慢地表达 |
| 节拍3 | 包子猪被按在案板中央，表情惊讶 |
| 节拍4 | 包子猪系好餐巾后，期待敲盘 |
| 节拍6 | 镇关西化身为紫肤泰坦，双手展开展现威严 |

**禁止表述**：

| 禁止类型 | 示例 |
|---------|------|
| 动作进展 | "包子猪被拉走" |
| 方向性动作 | "冲向包子" |
| 动态过程 | "正在运刀如风" |
| 时间序列 | "随后镇关西切臊子" |

**允许表述**：

| 允许类型 | 示例 |
|---------|------|
| 完成状态 | "包子猪系好餐巾后"（"后"=完成）|
| 静态姿态 | "鲁达站在案板前" |
| 悬浮状态 | "暗物质包子悬浮在案板上" |
| 转化结果 | "镇关西化身为紫肤泰坦" |

---

#### 步骤4：验证单帧合规性

**检查清单**：

- [ ] 没有"然后"、"接着"、"随后"等时间序列词
- [ ] 没有连续动作（多个动词连用）
- [ ] 没有方向性描述（"冲向"、"飞向"、"跑向"）
- [ ] 聚焦静态画面（姿态+表情+眼神）
- [ ] 是戏剧张力最强的帧

**违规示例及修复**：

| 违规 | 检测 | 修复 |
|------|------|------|
| "鲁达冲出滑铲接球" | 多个动作动词 | "鲁达滑铲姿势清晰" |
| "包子猪从惊恐变脸→期待" | "→"蒙太奇 | "包子猪希望眼神，期待敲盘" |
| "随后镇关西切臊子" | "随后"时间序列 | 去除"随后"和"切臊子" |

---

#### 步骤5：生成备选帧

**生成 JSON.alternative_frames**：

```json
{
  "alternative_frames": [
    {
      "frame_type": "起势",
      "role": "中景",
      "action": "鲁达张望，目光落在包子猪身上",
      "drama_tension_score": 1.2,
      "reason": "戏剧张力较弱，但动作启动点"
    },
    {
      "frame_type": "动作开始（抓猪）",
      "role": "中景",
      "action": "鲁达伸出手抓住包子猪",
      "drama_tension_score": 1.35,
      "reason": "戏剧张力强但不如按在案板帧"
    }
  ],
  "主帧": {
    "frame_type": "动作高潮帧",
    "role": "近景/特写",
    "action": "包子猪被按在案板中央，表情惊讶",
    "drama_tension_score": 1.4,
    "reason": "动作最高潮点，戏剧张力最强"
  }
}
```

**备选帧生成规则**：

1. **提取动作序列节点**：
   - 从完整描述中识别动作开始点
   - 记录动作进行的中间点
   - 记录动作结果点

2. **计算戏剧张力评分**：
   - **关键转折**（2.5-3.5）：咬腕、变身、爆炸
   - **情绪极值**（1.8-2.5）：期待眼神、惊恐、愤怒
   - **动作启动**（1.2-1.8）：发现、开始准备
   - **动作结果**（2.0-3.0）：完成、达成状态

3. **选择主帧**：
   - 选择戏剧张力评分最高的帧
   - 标记为主帧，写入 JSON.scene_description

4. **记录备选帧**：
   - 所有其他帧记录到 JSON.alternative_frames
   - 格式：{frame_type, role, action, drama_tension_score, reason}

## 处理示例

### 示例1：节拍3（动势组）

**输入完整描述**：

```
鲁达眯眼审视案板寻找破绽未果，四处张望。目光落在包子猪身上。
突然一只手抓住包子猪背带，从包子铺快速拉走，重重按在案板中央。
包子猪受惊表情。
```

**提取过程**：

1. **识别动作序列**：张望→抓猪→按在案板
2. **定位戏剧张力最强帧**："按在案板"（动作最高潮）
3. **生成单帧描述**："包子猪被按在案板，落在堆满精肉臊子的地方，表情惊讶"
4. **生成备选帧**：
   - 起势："鲁达张望"（1.2）
   - 动作开始："鲁达伸手抓住包子猪"（1.35）
   - 主帧："包子猪被按在案板，表情惊讶"（1.4）

### 示例2：节拍4（蒙太奇组）

**输入完整描述**：

```
鲁达："但这猪不满意！洒家要你把这几十斤肉全部装进一颗'包子'里，给它吃！"
包子猪看着案板上的猪肉漏出惊恐表情。但听到"给它吃"三个字，包子猪拼命挣扎动作瞬间静止。
0.1秒变脸：它极其熟练地掏出餐巾系在脖子上，双手亮出刀叉，眼神从惊恐变为极度期待，
兴奋用刀叉敲击盘子。音效"叮！"
```

**提取过程**：

1. **识别动作序列**：惊恐→听到给它吃→系餐巾→期待敲盘
2. **定位戏剧张力最强帧**："期待敲盘"（情绪极值）
3. **生成单帧描述**："包子猪系好餐巾、亮出刀叉后，眼神期待，兴奋用餐刀叉敲击盘子"
4. **生成备选帧**：
   - 起势："包子猪看着猪肉露出惊恐表情"（1.3）
   - 转折："听到'给它吃'三个字"（1.5）
   - 动作："掏出餐巾，系在脖子上，亮出刀叉"（1.7）
   - 主帧："包子猪期待敲盘"（1.9）

### 示例3：节拍5（蒙太奇组）

**输入完整描述**：

```
镇关西暴怒："欺人太甚！"鲁达怒喝："欺你又如何！"
镇关西大喝："给我……凝！！"镇关西周身爆发出红黑色的煞气，双手虚空对压。
案板上肉堆在气劲下被挤压变形，周围墨汁飞溅。气劲消散。
案板中央悬浮着拳头大小、漆黑如墨、周围缠绕红色闪电的"暗物质包子"。
它散发出沉重引力气场，案板因重力发出吱呀声。
```

**提取过程**：

1. **识别动作序列**：暴怒→周身爆发→肉堆变形→凝气消散→包子悬浮
2. **定位戏剧张力最强帧**："暗物质包子诞生"（物象生成，视觉高光）
3. **生成单帧描述**："暗物质包子诞生：案板中央悬浮，漆黑如墨，红色闪电缠绕，散发引力"
4. **生成备选帧**：
   - 起势："镇关西暴怒，两人对峙"（2.0）
   - 动作："镇关西周身红黑色煞气，双手虚空对压"（2.2）
   - 气劲："气劲消散"（2.3）
   - 主帧："暗物质包子诞生"（2.4）

## 与其他模块的对接

### 输出到

**JSON生成阶段（scene-breakdown.json 或 beat-board-full-list.json）**：

```json
{
  "grids": [
    {
      "frames": [
        {
          "scene_description": "主帧单帧画面描述",
          "frame_role": "动作高潮帧",
          "drama_tension_score": 1.4,
          "alternative_frames": [
            {
              "frame_type": "起势",
              "action": "起势帧动作",
              "drama_tension_score": 1.2,
              "reason": "戏剧张力较弱"
            }
          ]
        }
      ]
    }
  ]
}
```

### 与九宫格的对接

**九宫格生成使用 JSON.scene_description**：

```
beat-board-full-list.json
  ↓
grids[].frames[].scene_description
  ↓
九宫格格子的"画面描述"
```

### 与四宫格的对接

**四宫格使用 JSON.alternative_frames**：

```
beat-board-full-list.json
  ↓
grids[].frames[].alternative_frames
  ↓
四宫格展开：起势→动作→结果
```

## 质量保证

### 单帧合规率

**目标**：100%（所有格子的画面描述符合单帧原则）

### 违规检测

| 项目 | 检测方法 | 阈值 |
|------|---------|------|
| 连续动作动作 | 多个动作动词连用 | 0% |
| 时间序列词 | "然后"、"接着"、"随后" | 0% |
| 方向性动作 | "冲向"、"飞向"、"跑向" | 0% |
| 蒙太奇标识 | "→"、"→" | 0% |
| 格式合规 | **[关键帧]** 标签 | 100% |
