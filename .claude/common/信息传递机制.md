# 信息传递机制详细说明

> **【详细规范文档】**
>
> 本文件作为 AGENTS.md 的扩展说明，详细描述信息传递机制。
> 请参考 AGENTS.md 了解整体架构和流程。

 ---

[核心原则]

制片人负责**组织信息**，子 Agent 负责**路由到 SKILL**，SKILL 负责**读取模块并执行**。

```
制片人 → 子 Agent（路由）→ SKILL（流程控制）→ common/ 模块（具体逻辑）
```

[信息分类与传递]

| 信息类型 | 来源 | 传递方式 | 用途 |
|----------|------|----------|------|
| **项目配置** | 用户输入或 .agent-state.json | projectConfig 参数 | 视觉风格、目标媒介、画幅比例 |
| **剧本文件** | script/ 目录 | contextFiles 数组 | 生成内容的基础 |
| **前置产物** | outputs/ 目录 | contextFiles 数组 | 继承和引用 |
| **用户偏好** | 用户输入或 .agent-state.json | userPreferences 参数 | 角色设定、光线系统等 |
| **任务参数** | 指令解析 | taskSpecificData 参数 | 集数标识、特殊要求 |
| **快照信息** | .strategic-snapshots/ 目录 | snapshots 字段 | 项目回滚和版本管理 |
| **确认历史** | 用户确认流程 | confirmation_history 字段 | 记录用户决策历史 |

[阶段文件依赖关系]

```
剧本 (script/*.md)
    │
    ├─→ 节拍拆解 (outputs/beat-breakdown-{集数}.md)
    │       │
    │       ├─→ 九宫格 (outputs/beat-board-prompt-{集数}-board*.md)
    │       │       │
    │       │       └─→ 四宫格 (outputs/sequence-board-prompt-{集数}.md)
    │       │               │
    │       │               └─→ 动态提示词 (outputs/motion-prompt-{集数}.md)
    │       │
    └─→ 九宫格 (outputs/beat-board-prompt-{集数}-board*.md)  ← 需要节拍拆解

四宫格 (outputs/sequence-board-prompt-{集数}.md)
    │
    └─→ 动态提示词 (outputs/motion-prompt-{集数}.md)  ← 需要四宫格和九宫格
```

[制片人信息组织流程]

当收到用户指令时，制片人执行以下流程：

```
1. 解析指令
   - 提取指令类型（如 /breakdown）
   - 提取集数标识（如 ep01）
   - 提取特殊参数（如有）

2. 读取 .agent-state.json
   - 获取当前项目配置
   - 获取用户偏好
   - 获取前置产物列表

3. 确定需要的 context_files
   根据指令类型，确定需要传递给子 Agent 的文件：

   /breakdown:
   - script/*-{集数}.md (剧本文件)

   /beatboard:
   - outputs/beat-breakdown-{集数}.md (节拍拆解表)

   /sequence:
   - outputs/beat-board-prompt-{集数}.md (九宫格总览)
   - outputs/beat-board-prompt-{集数}-board*.md (九宫格分板)

   /motion:
   - outputs/sequence-board-prompt-{集数}.md (四宫格提示词)
   - outputs/beat-board-prompt-{集数}.md (九宫格总览)

   /review:
   - outputs/{产物类型}-{集数}.md (待审核文件)
   - outputs/{上游产物}-{集数}.md (上游产物，可选)

4. 调用子 Agent
   - 传递 projectConfig, userPreferences, contextFiles, taskSpecificData
   - 子 Agent 读取对应 SKILL，按流程执行

5. 汇总结果
   - 接收子 Agent 输出
   - 更新 .agent-state.json
   - 返回结果给用户

6. 【新增】执行用户确认（关键步骤）
   - 根据当前阶段确定确认点（CP1-CP5）
   - 展示标准确认UI（参考 user-confirmation-protocol.md）
   - 等待用户选择：通过 / 修改 / 重生成 / 回滚
   - 根据选择执行后续操作

7. 【新增】创建快照
    - 根据确认点类型创建对应快照（参考 snapshot-management.md）
    - 更新 .agent-state.json 记录快照信息
```

[信息传递规则]

1. **制片人 → 子 Agent**：通过 prompt 参数传递
2. **子 Agent → SKILL**：通过读取 SKILL.md 执行
3. **SKILL → 公共模块**：通过自动加载规则读取 common/ 文件
4. **状态更新**：通过 .agent-state.json 管理
