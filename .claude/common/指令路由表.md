# 指令路由表详细说明

[主指令]

| 指令 | 子 Agent | 任务 | 调用 SKILL | 说明 |
|------|----------|------|------------|------|
| `/breakdown [集数]` | storyboard-artist | 节拍拆解 | film-storyboard-skill | 从剧本生成节拍拆解表 |
| `/beatboard [集数]` | storyboard-artist | 九宫格提示词 | film-storyboard-skill | 基于节拍拆解生成多宫格 |
| `/sequence [集数]` | storyboard-artist | 四宫格提示词 | film-storyboard-skill | 基于九宫格生成四宫格 |
| `/motion [集数]` | animator | 动态提示词 | animator-skill | 基于四宫格生成动态提示词 |
| `/interactive` | 制片人 | 配置向导 | - | 引导用户配置项目 |
| `/status` | 制片人 | 状态检测（动态生成） | - | 从 router.js 动态生成状态报告 |
| `/snapshot [id]` | 制片人 | 快照回滚 | - | 恢复到指定快照 |
| `/creative [指令]` | storyboard-artist | 添加创意 | film-storyboard-skill | 在当前产出中添加创意 |

[审核指令]

| 指令 | 子 Agent | 任务 | 调用 SKILL |
|------|----------|------|------------|
| `/review breakdown [集数]` | director | 审核节拍拆解 | storyboard-review-skill |
| `/review beatboard [集数]` | director | 审核九宫格 | storyboard-review-skill |
| `/review sequence [集数]` | director | 审核四宫格 | storyboard-review-skill |
| `/review motion [集数]` | director | 审核动态提示词 | storyboard-review-skill |

[指令解析规则]

1. 提取指令部分（如 `/breakdown`）
2. 提取集数标识（如 `ep01`）
3. 提取特殊参数（如有）

[路由优先级]

1. 先匹配主指令表
2. 再匹配审核指令表
3. 最后匹配管理指令表

[指令示例]

```bash
# 节拍拆解
/breakdown ep01

# 九宫格提示词（使用默认集数 ep01）
/beatboard

# 四宫格提示词（指定集数）
/sequence ep02

# 审核九宫格
/review beatboard ep01

# 查看状态
/status

# 快照管理
/snapshot list              # 查看快照列表
/snapshot preview <id>      # 预览快照
/snapshot rollback <id>     # 回滚到快照
/snapshot create "备注"     # 创建手动快照
```

[快照管理指令]

详见 `common/snapshot-management.md`

| 指令 | 说明 |
|------|------|
| `/snapshot` | 显示快照管理菜单 |
| `/snapshot list` | 列出所有可用快照 |
| `/snapshot preview <id>` | 预览指定快照 |
| `/snapshot rollback <id>` | 回滚到指定快照 |
| `/snapshot create <备注>` | 创建手动快照 |
| `/snapshot delete <id>` | 删除手动快照（仅手动） |
