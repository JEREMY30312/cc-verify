### 新增字段说明（蓝图A）

- **核心策略**：节拍的主导拍摄框架，必须且唯一。格式：[策略类型:关键描述]
  - 策略类型：单镜、动势组、蒙太奇组、长镜头
  - 示例：[单镜:两人对峙]、[动势组:冲向门口]、[蒙太奇组:闪回记忆]、[长镜头:走廊追逐]

- **辅助信息**：复杂节拍的内部变化描述，可选。格式：[内含:变化描述]
  - 触发条件：包含转折词、时长>8秒、情绪剧烈变化、高强度感官锚点
  - 示例：[内含:对话铺垫→突然爆发]

- **关键信息点**：JSON格式的拆分锚点数组，为四宫格提供具体拆分点
  - 包含类型（如"动作开始"、"起势"）和内容
  - 根据策略类型提取不同类型的关键点

- **复杂度评分**：1.0-5.0的评分，基于内容复杂性
  - 算法：基础分 + 0.3×(关键点数量) + 0.5×(如有辅助信息) + 0.2×(潜台词数量) + 0.5×(感官锚点数量)

- **建议格子数**：1-4格的建议，指导四宫格分配
  - 算法：min(max(round(复杂度评分), 1), 4)

  - **拆解方案**（Phase 2.4动态化）：根据核心策略类型动态生成的拆分方案描述
     - **[单镜]**："单镜头固定机位，重点表现{情绪基调}"
       - 示例："单镜头固定机位，重点表现期待"
       - 九宫格关键帧: 整个单镜头画面
       - Sequence 四宫格拆分: 无需拆分 [strategy: 单镜, grid_count: 1]
     - **[动势组]**："2格动势分解，戏剧张力最强帧: {起势或落幅}, 推荐4格拆分: 起势→落幅"

## 角色设定使用规则（方案D）

### 访问角色设定

**格式**：
```
scene_breakdown_json["character_settings"][角色名][风格名][字段名]
```

**示例**：
```python
# 获取鲁达在国潮动漫风格下的头饰
luda_headwear = scene_breakdown_json["character_settings"]["鲁达"]["国潮动漫风格"]["headwear"]
# → "暗青灰色芝麻罗纱头巾，哑光无反光质感"
```

### 节拍风格检测

**算法**：
```python
def get_style_for_beat(beat_number):
    """根据节拍号获取当前应使用的风格"""
    for transition in style_transitions:
        if beat_number >= transition["beat"]:
            return transition["to"]
    return style_transitions[0]["from"] if style_transitions else "默认风格"
```

### 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| physical | string | ✅ | 身形描述 |
| headwear | string/null | ⚪ | 头饰描述 |
| upper_body | string | ✅ | 上装描述 |
| waist | string/null | ⚪ | 腰饰描述 |
| footwear | string/null | ⚪ | 鞋履描述 |
| accessories | string/null | ⚪ | 配饰描述 |
| skin | string/null | ⚪ | 皮肤描述 |
| rendering | string | ✅ | 渲染风格描述 |

注意：null值表示该角色/风格组合下该字段不适用
