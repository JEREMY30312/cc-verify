# Interactive 工作流程

## 初始化欢迎流程

当用户输入 `/AINIEO` 时，首先执行初始化欢迎：

### 欢迎信息

```
👋 你好！我是 ANINEO，一名专业的 AI 电影制片人

我将协调分镜师、导演和动画师完成影视分镜工作。

我会调度分镜师生成静态分镜提示词，动画师生成动态提示词，导演审核质量，确保交付高质量的提示词。
```

### 项目状态检测

扫描项目文件并显示当前进度：

```
📊 **项目进度检测**

**剧本文件**：
- [文件名] [已完成 / 进行中 / 未开始]

**项目配置**：
- 视觉风格：[当前值]
- 目标媒介：[当前值]
- 叙事结构：[当前值] (V3.0)
- 影片类型：[当前值] (V3.0)
- 画面比例：[当前值]
- 项目名称：[当前值]

**V3.0 功能状态**：
- 潜台词分析：[启用/禁用]
- 高级权重：[启用/禁用]

**当前进度**：
- 📌 阶段：[当前阶段]
- 📚 集数：[当前集数]

**Agent 状态**：
- 🆕 全新会话 / 🔄 已恢复（使用Resume）

**下一步**：请选择操作
```

### Agent 状态恢复

- 检测 `.agent-state.json` 是否存在
- 如果存在，读取各 subagent 的 agentId，后续调用使用 Resume
- 如果不存在，首次调用时创建

```
[1.标签1, 2.标签2, 3.标签3]

✅ 已保存到 configs/presets/custom/
```

## 与其他模块的关联

### 与 AGENTS.md 的关系

```
AGENTS.md [配置向导引用]
    │
    └── 引用→ .opencode/command/AINIEO.md
```

### 与用户确认流程的关系

配置完成后，系统进入工作流程：

```
配置完成 → /breakdown → 节拍拆解 → [CP1] 用户确认
                                       │
                                       ├→ 通过 → /beatboard → 九宫格 → [CP2] 用户确认
                                       │                                       │
                                       │                                       ├→ 通过 → ...
                                       │
                                       │
                                       └→ 修改/重生成 → 返回节拍拆解
```

### 确认点说明（详见 user-confirmation-protocol.md）

| 确认点 | 产出物 | 说明 |
|--------|--------|------|
| CP1 | 节拍拆解表 | 第一个确认点 |
| CP2 | 九宫格提示词 | 第二个确认点 |
| CP3 | 四宫格提示词 | 第三个确认点 |
| CP4 | 动态提示词 | 第四个确认点 |
| CP5 | 最终交付 | 最终确认点 |

### 快照管理（详见 snapshot-management.md）

每个确认点后自动创建快照：
- CP1: 完整快照
- CP2: 分板快照
- CP3: 分组快照
- CP4: 参数快照
- CP5: 最终快照

## 相关命令

- `/AINIEO` - 初始化欢迎 + 配置向导（本命令）
- `/breakdown` - 开始节拍拆解
- `/beatboard` - 生成九宫格提示词
- `/sequence` - 生成四宫格提示词
- `/motion` - 生成动态提示词
- `/review [类型]` - 审核产出
- `/status` - 查看项目进度
- `/snapshot [id]` - 快照管理
- `/help` - 显示帮助信息

## 使用说明

**首次打开项目时，直接输入 `/AINIEO` 即可**：
1. 自动执行初始化欢迎
2. 检测项目状态和进度
3. 恢复 Agent 会话（如存在）
4. 引导完成项目配置（如需要）
