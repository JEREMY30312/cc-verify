# 异常处理与容错机制 Exception Handler

[功能]
    定义各类异常情况的处理方法

[关键帧不足处理]
    场景：入选关键帧少于每板最小要求

    处理流程：
    1. 系统检测到关键帧不足
    2. 提示导演关键帧数量
    3. 导演选择：
       a) 使用空白占位格
       b) 提升中级关键帧等级
       c) 手动添加补充关键帧
    4. 记录处理结果

[布局不匹配处理]
    场景：关键帧类型与布局类型不匹配

    处理流程：
    1. 系统检测到布局不匹配
    2. 提供调整建议（如将动作序列改为1×3线性网格）
    3. 导演选择：
       a) 接受系统建议
       b) 手动调整关键帧类型
       c) 保持当前布局（需说明理由）
    4. 记录处理结果

[导演决策冲突处理]
    场景：导演选择与系统建议严重冲突

    处理流程：
    1. 系统检测到严重冲突
    2. 计算风险评估分数
    3. 提供风险提示给导演
    4. 要求导演确认并说明理由
    5. 记录冲突原因和导演决定

[审核不通过处理]
    场景：用户审核不通过产出

    处理流程：
    1. 记录审核不通过的具体原因
    2. 分类问题严重程度（关键/重要/次要）
    3. 生成修改需求清单
    4. 启动修订流程
    5. 重新生成后再次提交审核

[系统错误处理]
    场景：系统执行过程中发生错误

    处理流程：
    1. 捕获错误信息
    2. 记录错误日志
    3. 提供错误摘要给用户
    4. 尝试恢复到最近保存的版本
    5. 提示用户如何手动恢复

[使用方式]
    直接读取此文件，按流程处理各类异常情况
