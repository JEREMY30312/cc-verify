## 九宫格提示词

### ⚠️ 九宫格核心原则：每格=一个关键帧 [V4.1-CRITICAL]

**设计定位**：
- 九宫格是 **关键帧选集（Keyframe Selection）**，而非动作拆分子母
- 每格必须只包含 **一个** 画面场景（Single Frame）
- 动作拆分由 Sequence 阶段的四宫格完成

**禁止模式**：
- ❌ 一格内包含多个视角（如"正反打A角"+"正反打B角"）
- ❌ 一格内包含连续动作（如"起势"+"落幅"）
- ❌ 使用"→"或"-"箭头连接多个画面
- ❌ 使用"[主格-xxx]+[副格-xxx]"格式

**正确模式**：
- ✅ 每格只保留戏剧张力最强的单帧（基于 3 因子算法）
- ✅ 其他拆分帧记录到 JSON 的 `alternative_frames`（仅类型+原因）
- ✅ 使用明确的标签如"[关键帧-镇关西特写]"标注单帧内容

**戏剧张力算法（3因子）**：
```
戏剧张力 = 视觉冲击权重 × 情绪转折权重 × 叙事重要性权重

| 因子 | 权重范围 | 说明 |
|------|----------|------|
| 视觉冲击权重 | 0.5-1.0 | A角/起势: 1.0（主动动态）<br>B角/落幅: 0.8（被动结果） |
| 情绪转折权重 | 0.5-1.5 | 表情变化: 1.2<br>静态维持: 0.5 |
| 叙事重要性权重 | 0.8-1.2 | 诱因事件: 1.2<br>过渡节拍: 0.8 |
```

**决策示例（节拍2：正反打）**：
- 镇关西特写: 1.0 × 1.2 × 1.2 = 1.44（当选）
- 鲁达反应: 0.8 × 0.5 × 0.8 = 0.32（记录到 alternative_frames）

**决策示例（节拍5：动势组）**：
- 起势（抓猪）: 1.0 × 1.0 × 1.2 = 1.2（当选）
- 落幅（按案板）: 0.8 × 0.8 × 1.0 = 0.64（记录到 alternative_frames）

**对比示例**：

| 违规示例（❌） | 正确示例（✅） |
|------------------|----------------|
| 格02: [主格-正反打A角] + [副格-正反打B角]<br>(同时包含两个视角) | 格02: [关键帧-镇关西特写]<br>(只保留戏剧张力更强的单帧) |
| 格05: [主格-起势] + [副格-落幅]<br>(同时包含两个动作) | 格05: [关键帧-起势抓猪瞬间]<br>(只保留更具戏剧性的单帧) |
| 格XX: 场景A → 场景B → 场景C<br>(使用箭头连接) | 格XX: [关键帧-场景B]<br>(只保留戏剧张力最强的B) |

**历史文件处理**：
- ⚠️ 旧版本生成的九宫格（V4.0及之前）可能包含"一格多画面"
- 识别标记：画面描述中包含"+"、"→"或多个"[xxx-xxx]"标签
- ⚠️ 新版本（V4.1+）严格遵循单帧原则

```

### 字段说明 [V4.1新增]

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| is_single_frame | boolean | ✅ | 强制为 true（确保单帧） |
| drama_tension_score | number | ✅ | 戏剧张力评分（0-2.0），3因子算法计算 |
| selected_frame_type | string | ✅ | 选择类型（primary/secondary） |
| frame_role | string | ⚪ | 角色名称，如无则为 null |
| frame_action | string | ⚪ | 动作描述，如无则为 null |
| alternative_frames | array | ⚪ | 未选择的拆分帧（方案B：类型+reason） |
| quality_metrics.single_frame_complidity | number | ✅ | 单帧合规性（应为 100） |
| quality_metrics.alternative_frames_recorded | number | ✅ | 记录的未选择帧数量 |

### alternative_frames 数组说明（方案B）

每个 alternative_frames 条目仅包含类型和原因，不存储完整描述：

| 字段 | 类型 | 说明 |
|------|------|------|
| frame_type | string | secondary 或 tertiary |
| role | string | 角色名称 |
| action | string | 简要动作描述（1-2句话） |
| drama_tension_score | number | 戏剧张力评分 |
| reason | string | 选择原因说明 |

**方案B优势**：
- ✅ 文件体积小（减少10-30%）
- ✅ 无冗余（避免重复存储）
- ✅ 灵活性强（Sequence阶段可重新生成）
- ✅ 符合V5.0精简优化方向
}
```

### 输出说明

- **grids数组**：9个格子的完整数据
- **waterfall_shots**：水下镜头映射（不显示在板中的镜头）
- **inheritance字段**：从scene-breakdown继承的数据
