## 四宫格完整性校验（第三阶段新增）

### 目的
防止四宫格阶段漏掉关键剧情或台词，确保叙事完整性。

### 【V5.0新增】字段完整性检查

**检查逻辑**：
- 检查 `scene-breakdown-{集数}.json` 中每个节拍的 `source_text` 字段
- 验证字段是否存在且非空
- 验证内容是否与剧本原文一致（抽样对比）
- 验证与 beat-breakdown 中的"场景描述"逻辑一致

**通过标准**：
- ✅ 所有节拍都有 source_text 字段且非空：通过
- ⚠️ 部分节拍缺少 source_text：警告，必须补充
- ❌ source_text 与剧本原文不符：不通过，必须修正

**失败处理**：
- 如 source_text 缺失 → 标记为 [MISSING_SOURCE]，要求重新生成节拍拆解
- 如 source_text 与剧本不符 → 标记为 [MISMATCH]，要求核对剧本原文
- 如场景描述与原文逻辑不符 → 标记为 [INCONSISTENT]，要求修正描述

**检查项**：
    - [ ] source_text 字段存在性：每个节拍都有该字段
    - [ ] source_text 内容非空：字段值不为空字符串
    - [ ] 原文一致性：抽样检查与剧本原文匹配度 ≥ 95%
    - [ ] 逻辑一致性：场景描述与原文内容逻辑相符

### 台词落地率检查

**检查逻辑**：
- 提取剧本中所有标记为"重要对话"的台词（通过节拍拆解表识别）
- 检查每句台词是否能在四宫格的"画面描述"或"潜台词"字段中找到对应
- 计算覆盖率：已覆盖台词数 / 总台词数

**通过标准**：
- ✅ 覆盖率 ≥ 95%：通过
- ⚠️ 90% ≤ 覆盖率 < 95%：警告，建议补充
- ❌ 覆盖率 < 90%：不通过，必须补充

**处理方式**：
遗漏台词："欺人太甚！" (节拍8)
建议补充到：四宫格第3组，镜头2（可在画面描述中加入该台词）

### 动作完整度检查

**检查逻辑**：
- 识别所有标记为"重大动作"或带有[蒙太奇组]标签的节拍
- 检查这些节拍在四宫格中是否至少占用了2个以上的格子
- 对于[蒙太奇组]，检查是否已展开为"起势-动作-结果"三段式序列

**通过标准**：
- 重大动作节拍：必须占用 ≥ 2 个格子
- 蒙太奇组节拍：必须展开为完整的3段式序列
- 动势组节拍：必须包含起幅和落幅两帧

### 视觉连贯性检查

**检查逻辑**：
- 检查同一组（Sequence Board）内的光影方向是否突变
- 检查角色资产是否一致（如：上一格拿枪，下一格空手）
- 检查空间关系是否连贯（如：门的位置、角色的相对位置）

**检查项**：
- [ ] 光影方向一致性：检查所有镜头的光源方向是否统一
- [ ] 角色资产连续性：检查角色手中的道具、服装状态是否连贯
- [ ] 空间布局一致性：检查场景布局、家具位置是否保持不变

### 回溯校验清单

**台词落地率**：
    - [ ] 检查剧本中每一句带情绪标注的台词
    - [ ] 是否能在四宫格的"画面描述"或"潜台词"中找到对应
    - [ ] 若无，标记为 [MISSING]，并建议补充位置

**动作完整度**：
    - [ ] 检查所有 [蒙太奇组] 标签的节拍
    - [ ] 确认其在四宫格中是否已展开为"起势-动作-结果"序列
    - [ ] 检查所有 [动势组] 是否包含起幅和落幅两帧

**视觉连贯性**：
    - [ ] 检查同一组内的光影方向是否突变（如：上一镜左侧光，下一镜右侧光）
    - [ ] 检查角色资产是否丢失（如：武器、关键道具）
    - [ ] 检查场景布局是否一致（如：家具位置、门窗方向）
