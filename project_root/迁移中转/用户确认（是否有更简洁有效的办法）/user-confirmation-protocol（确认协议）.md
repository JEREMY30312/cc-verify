# 用户战略确认协议

> **【详细规范文档】**
>
> 本文件作为 AGENTS.md 的扩展说明，详细描述用户确认流程。
> 请参考 AGENTS.md 了解整体架构和流程。

 ---

## 核心原则

每个阶段完成后必须执行用户战略确认，确保用户对产出质量有完全控制权。

## 1. 确认节点系统

### 1.1 五个核心确认点

```
剧本输入
↓
[CP1] 节拍拆解确认
    ├→ 通过 → [CP2] 九宫格确认
    │ ├→ 通过 → [CP3] 四宫格确认
    │ │ ├→ 通过 → [CP4] 动态提示词确认
    │ │ │ ├→ 通过 → [CP5] 最终交付确认 → 完成
    │ │ │ └→ 修改/重生成 → 返回CP4
    │ │ └→ 修改/重生成 → 返回CP3
    │ └→ 修改/重生成 → 返回CP2
    └→ 修改/重生成 → 返回CP1
```

### 1.2 确认点配置

| 确认点 | 产出物 | 影响范围 | 快照类型 |
|--------|--------|----------|----------|
| CP1 | 节拍拆解表 | 影响后续所有阶段 | 完整快照 |
| CP2 | 九宫格提示词 | 影响四宫格和动态词 | 分板快照 |
| CP3 | 四宫格提示词 | 影响动态提示词 | 分组快照 |
| CP4 | 动态提示词 | 仅影响本阶段 | 参数快照 |
| CP5 | 所有产出文件 | 最终交付锁定 | 最终快照 |

## 2. 用户操作接口

### 2.1 文本编辑方式

| 方式 | 说明 | 示例 |
|------|------|------|
| 内联编辑 | 使用标记修改内容 | `// MODIFY START ... // MODIFY END` |
| 区块编辑 | `/edit beat 3` | 弹出编辑框修改 |
| 批注编辑 | 添加 `<!-- USER_NOTE: ... -->` | 标记修改建议 |

### 2.2 创意指令

格式：`/creative <类型> <内容> [参数]`

| 类型 | 示例 |
|------|------|
| 风格融合 | `/creative style fusion 赛博朋克+水墨风` |
| 参考指定 | `/creative reference 《蜘蛛侠：平行宇宙》 抽帧效果` |
| 角色变体 | `/creative character variant 鲁智深 机甲版` |
| 氛围调整 | `/creative atmosphere 紧张感 高` |
| 技术特效 | `/creative effect 镜头光晕 全局` |

### 2.3 批处理操作

- [全部重生成]
- [按类型重生成]
- [单节拍编辑]

### 2.4 操作选项

| 操作 | 功能 |
|------|------|
| **通过** | 确认当前产出，进入下一阶段 |
| **修改** | 指明需要修改的内容 |
| **重生成** | 重新生成当前产出 |
| **回滚** | 恢复到之前的版本 |

### 2.5 修改影响分析

| 影响程度 | 阈值 | 说明 |
|----------|------|------|
| 高影响 | >50%元素 | 可能导致后续阶段重做 |
| 中影响 | >20%元素 | 需要后续阶段调整 |
| 低影响 | >5%元素 | 后续阶段微调即可 |
| 无影响 | ≤5%元素 | 不影响后续阶段 |

## 4. 确认阶段详情

### CP1: 节拍拆解确认
- 产出物：节拍拆解表
- 质量评分：4.5/5
- 操作：通过 / 修改 / 重生成 / 回滚

### CP2: 九宫格确认
- 产出物：九宫格提示词
- 质量评分：4.3/5
- 操作：通过 / 修改 / 重生成 / 回滚

### CP3: 四宫格确认
- 产出物：四宫格提示词
- 质量评分：4.4/5
- 操作：通过 / 修改 / 重生成 / 回滚

### CP4: 动态提示词确认
- 产出物：动态提示词
- 质量评分：4.2/5
- 操作：通过 / 修改 / 重生成 / 回滚

### CP5: 最终交付确认
- 产出物：所有产出文件
- 操作：确认交付 / 返回修改

## 5. 与其他模块的集成

### 5.1 与审核流程的关系

```
产出生成 → director审核 → 用户确认 → 进入下一阶段
           ↓              ↓
        通过/不通过    通过/修改/重生成/回滚
```

### 5.2 与快照管理的关系

详见 `common/snapshot-management.md`

每个确认点自动创建对应类型的快照，支持回滚到任意历史版本。
