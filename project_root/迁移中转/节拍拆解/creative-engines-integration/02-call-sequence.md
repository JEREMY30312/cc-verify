# 引擎调用时序规范

## 2.1 完整工作流调用时序

**节拍拆解阶段**
- 并行调用：蒙太奇逻辑引擎 + 影视联想引擎
- 条件调用：检测到通感描述 → 通感视觉化引擎
- 输出整合：三个引擎输出整合到节拍拆解表

**多宫格生成阶段**
- 每帧前调用：蒙太奇逻辑引擎（主导）
- 每场景调用：影视联想引擎（参考）
- 按需调用：通感视觉化引擎（严格条件）
- 环境构造：自动应用环境深度规则

**四宫格生成阶段**
- 继承调用：继承多宫格的引擎输出
- 细化调用：蒙太奇逻辑引擎微调动作分解
- 补充调用：影视联想引擎提供细节参考
- 禁用调用：通感引擎（避免过度特效）

## 2.2 调用频率控制规则

| 引擎类型 | 最大调用频率 | 缓存策略 | 降级条件 |
|----------|--------------|----------|----------|
| 蒙太奇逻辑 | 每关键帧1次 | 相似场景缓存5分钟 | 置信度<60%时简化 |
| 影视联想 | 每场景1次 | 相同风格缓存10分钟 | 匹配度<70%时忽略 |
| 通感视觉化 | 每感官事件1次 | 不缓存 | 强度<7时自动忽略 |

## 2.3 性能优化策略

### 批量处理
- 相似节拍合并分析
- 连续场景批量检索
- 引擎输出预生成

### 缓存机制
缓存键生成规则：
- 蒙太奇逻辑：剧本哈希 + 情绪基调 + 节拍类型
- 影视联想：风格关键词 + 情绪列表 + 视觉偏好
- 通感视觉化：通感语句哈希 + 强度等级

缓存时间：
- 高置信度(≥80%)：10分钟
- 中置信度(60-79%)：5分钟
- 低置信度(<60%)：不缓存

### 降级策略
- 系统负载高时，降低引擎调用深度
- 时间紧迫时，使用简化分析模式
- 用户明确要求时，跳过非关键引擎
