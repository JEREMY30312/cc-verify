# 引擎输出融合规则

## 3.1 优先级体系
1. 第一优先级：用户明确指令
2. 第二优先级：蒙太奇逻辑引擎（叙事逻辑）
3. 第三优先级：影视联想引擎（视觉风格）
4. 第四优先级：通感视觉化引擎（特效增强）
5. 第五优先级：环境深度构造规则（细节填充）

## 3.2 冲突解决流程
检测冲突 → 分析冲突类型 → 应用解决规则 → 标记冲突项 → 导演/用户裁决

**冲突检测标准：**
- 不同引擎对同一元素建议相反方案
- 引擎建议与基础规则矛盾
- 引擎建议与资产库定义冲突
- 引擎建议可能导致连贯性问题

## 3.3 冲突解决规则表

| 冲突类型 | 涉及引擎 | 默认解决方案 | 需要人工裁决 |
|----------|----------|--------------|--------------|
| 镜头数量冲突 | 蒙太奇 vs 基础规则 | 服从蒙太奇逻辑 | 当差异>3个镜头 |
| 风格选择冲突 | 影视联想 vs 用户偏好 | 服从用户偏好 | 当用户无明确偏好 |
| 特效强度冲突 | 通感 vs 风格统一 | 限制特效范围 | 当特效可能破坏叙事 |
| 环境细节冲突 | 环境构造 vs 画面简洁 | 分层控制细节 | 当细节可能干扰焦点 |
| 角色表现冲突 | 所有引擎 vs 资产库 | 服从资产库定义 | 当引擎建议更符合剧情 |

## 3.4 冲突标记格式

```markdown
⚠️ **引擎冲突标记** - [冲突ID]

**冲突位置**
- 文件：[文件名]
- 位置：[具体行/格/场景]
- 影响范围：[直接/间接影响]

**冲突详情**
- 引擎A：[引擎名称] 建议：[具体建议]
  理由：[引擎提供的理由]
  置信度：[0-100]%
  
- 引擎B：[引擎名称] 建议：[具体建议]
  理由：[引擎提供的理由]
  置信度：[0-100]%

- 其他相关因素：[资产库/用户指令/基础规则]

**系统分析**
- 冲突类型：[分类]
- 影响评估：高/中/低
- 默认解决方案：[系统推荐]
- 推荐理由：[分析]

**处理选项**
- [ ] 采用引擎A建议
- [ ] 采用引擎B建议
- [ ] 采用系统推荐方案
- [ ] 用户自定义方案：[ ]

**裁决记录**
- 裁决人：[系统/导演/用户]
- 裁决时间：[时间戳]
- 最终方案：[描述]
- 执行结果：[PASS/FAIL]
```
