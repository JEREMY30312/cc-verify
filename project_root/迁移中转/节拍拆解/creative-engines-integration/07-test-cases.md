# 集成测试用例

## 7.1 标准测试用例

```yaml
测试用例1：标准动作场景
输入:
  剧本: "A挥拳击倒B"
  权重: 高
  风格: 真人写实
预期输出:
  蒙太奇逻辑: 起势-动作-结果3帧
  影视联想: 参考《谍影重重》手持摄影
  通感视觉化: 不触发
  环境构造: 中景为主，前景碎片

测试用例2：情绪转折场景
输入:
  剧本: "她看到信后震惊落泪"
  权重: 中
  风格: 剧情片
预期输出:
  蒙太奇逻辑: 刺激源-反应-交互3帧
  影视联想: 参考《天使爱美丽》暖色调
  通感视觉化: 不触发
  环境构造: 特写为主，浅景深

测试用例3：通感场景
输入:
  剧本: "爆炸声震耳欲聋"
  权重: 高
  风格: 动作片
预期输出:
  蒙太奇逻辑: 爆炸瞬间1帧
  影视联想: 参考《拯救大兵瑞恩》纪实感
  通感视觉化: 触发，镜头抖动+冲击波
  环境构造: 广角，多层尘土
```

## 7.2 边界测试用例

```yaml
边界测试1：极低权重场景
输入: 日常对话，权重: 低
预期: 蒙太奇逻辑简化，其他引擎不调用

边界测试2：风格冲突场景
输入: 科幻剧本，指定复古胶片风格
预期: 影视联想引擎提供融合方案

边界测试3：多引擎冲突场景
输入: 复杂动作+强通感+特定风格
预期: 正确标记冲突，提供解决建议
```
