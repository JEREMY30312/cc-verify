# 多结构叙事模板定义

## 概述

本文件定义了 ANINEO V3.0 系统支持的多结构叙事模板，打破单一三幕式结构，提供四种叙事模板以适应不同类型的影视内容。

## 配置字段定义

### 项目配置扩展字段

在 `.agent-state.json` 的 `projectConfig` 中新增以下字段：

```json
{
  "narrative_structure": "classic_three_act",  // 默认值
  "structure_config": {
    "compatibility_mode": "auto_migrate",      // 兼容性模式
    "custom_weights": {},                      // 自定义权重配置
    "validation_strictness": "medium"          // 验证严格度
  }
}
```

### 字段说明

| 字段 | 类型 | 可选值 | 默认值 | 说明 |
|------|------|--------|--------|------|
| `narrative_structure` | string | `classic_three_act`, `hero_journey`, `kishotenketsu`, `micro_drama_loop` | `classic_three_act` | 叙事结构类型 |
| `structure_config.compatibility_mode` | string | `auto_migrate`, `strict_validation`, `legacy_support` | `auto_migrate` | 兼容性处理模式 |
| `structure_config.custom_weights` | object | 自定义权重映射 | `{}` | 自定义权重配置 |
| `structure_config.validation_strictness` | string | `low`, `medium`, `high` | `medium` | 验证严格度 |

## 结构模板定义

### 1. 经典三幕式 (Classic Three-Act)

**标识符**: `classic_three_act`

**适用场景**: 传统电影、电视剧、标准叙事内容

**结构特征**:
- **节拍数量**: 8-12个
- **三幕比例**: 
  - 第一幕: 15-25% (铺垫、激励事件)
  - 第二幕: 50-70% (发展、转折、中点)
  - 第三幕: 15-25% (高潮、结局)
- **关键帧分布**:
  - 特级关键帧 (🔴): 1-3个 (激励事件、中点、高潮)
  - 高级关键帧 (🟠): 2-4个 (转折点、危机)
  - 中级关键帧 (🟡): 3-5个 (发展、结局)

**权重计算规则**:
- 沿用现有权重系统: `综合权重 = 表现权重 × 基重`
- 表现权重范围: 0.2-2.0
- 基重范围: 1.0-2.0

**向后兼容性**: 完全兼容现有系统，作为默认结构

### 2. 英雄之旅 (Hero's Journey)

**标识符**: `hero_journey`

**适用场景**: 奇幻冒险、成长故事、史诗叙事

**结构特征**:
- **节拍数量**: 12个 (标准12步)
- **阶段划分**:
  - 出发阶段 (1-4): 平凡世界 → 冒险召唤 → 拒绝召唤 → 导师出现
  - 启蒙阶段 (5-8): 跨越第一道门槛 → 考验、盟友、敌人 → 接近最深的洞穴 → 严峻考验
  - 归来阶段 (9-12): 报酬 → 返回的路 → 复活 → 携万能药回归
- **关键帧分布**:
  - 特级关键帧 (🔴): 节拍5、8、11 (跨越门槛、严峻考验、复活)
  - 高级关键帧 (🟠): 节拍2、4、7、10 (冒险召唤、导师出现、接近洞穴、返回的路)
  - 中级关键帧 (🟡): 其余节拍

**权重计算规则**:
- **事件类型权重** (新系统): 0.1-4.0
- **叙事功能基重**: 1.0-2.5 (扩展范围)
- **类型修正系数**: 根据事件类型调整 (动作: ×1.2, 情感: ×1.1, 对话: ×0.9)

**特殊要求**:
- 必须包含"导师"角色
- 必须包含"跨越门槛"的仪式性场景
- 必须包含"携万能药回归"的升华结局

### 3. 起承转合 (Kishōtenketsu)

**标识符**: `kishotenketsu`

**适用场景**: 东方叙事、文艺片、情感细腻的内容

**结构特征**:
- **节拍数量**: 4个 (标准四段式)
- **阶段定义**:
  - 起 (Ki): 引入主题和角色 (25%)
  - 承 (Shō): 发展和深化主题 (25%)
  - 转 (Ten): 意外转折或新视角 (25%)
  - 合 (Ketsu): 融合与结论 (25%)
- **关键帧分布**:
  - 特级关键帧 (🔴): 节拍3 (转)
  - 高级关键帧 (🟠): 节拍4 (合)
  - 中级关键帧 (🟡): 节拍1、2 (起、承)

**权重计算规则**:
- **情感密度权重**: 0.5-3.0 (强调情感表达)
- **视觉诗意系数**: ×1.0-1.8 (根据画面美感调整)
- **留白权重**: 对话和静默场景有特殊权重计算

**特殊要求**:
- 强调画面构图和视觉诗意
- 允许更长的情感铺垫
- 转折点必须自然而不突兀
- 结论要有余韵和思考空间

### 4. 多巴胺闭环 (Micro-Drama Loop)

**标识符**: `micro_drama_loop`

**适用场景**: 短视频、短剧、社交媒体内容

**结构特征**:
- **节拍数量**: 4-6个 (根据时长调整)
- **时间结构**:
  - 0-15秒: 钩子 (强烈吸引)
  - 15-45秒: 发展 (快速推进)
  - 45-75秒: 转折 (意外反转)
  - 75-120秒: 闭环 (满足或悬念)
- **关键帧分布**:
  - 特级关键帧 (🔴): 钩子和转折点
  - 高级关键帧 (🟠): 每45秒一个高潮点
  - 中级关键帧 (🟡): 连接性内容

**权重计算规则**:
- **注意力权重**: 1.0-3.0 (根据吸引程度)
- **反转系数**: 转折点权重 ×1.5
- **闭环满意度**: 结局权重根据情感满足度调整

**特殊要求**:
- 开头3秒必须有强烈视觉钩子
- 每45秒必须有一个情感高潮
- 结尾必须有明确CTA或悬念
- 节奏必须快速紧凑

## 集成说明

### 1. 系统集成点

**节拍分析器 (`beat-analyzer.md`)**:
- 根据 `narrative_structure` 选择对应的分析逻辑
- 应用结构特定的节拍数量、比例和权重规则
- 输出结构特定的验证结果

**质量检查 (`quality-check.md`)**:
- 添加结构特定的检查项
- 验证结构约束条件
- 输出结构兼容性报告

**导演决策 (`director-decision.md`)**:
- 提供结构选择建议
- 验证结构适用性
- 输出结构优化建议

### 2. 迁移机制

**自动迁移** (默认模式):
- 现有项目自动设置为 `classic_three_act`
- 保持现有节拍拆解不变
- 仅更新配置字段

**严格验证**:
- 验证现有内容是否符合新结构
- 提供迁移建议
- 需要用户确认

**传统支持**:
- 完全保持现有行为
- 仅记录结构类型
- 不应用新规则

### 3. 验证流程

**结构适用性验证**:
1. 分析剧本内容特征
2. 匹配最适合的结构类型
3. 输出匹配度和建议

**节拍分布验证**:
1. 验证节拍数量是否符合结构要求
2. 验证关键帧分布是否符合结构特征
3. 输出验证报告和修正建议

**权重计算验证**:
1. 验证权重计算是否符合结构规则
2. 检查权重范围和分布
3. 输出权重优化建议

## 测试用例

### 测试1: 英雄之旅验证
- **输入**: 奇幻冒险剧本，选择 `hero_journey` 模板
- **验证**: 检查是否生成12个节拍，节拍5、8、11为🔴特级关键帧
- **预期**: 节拍4包含"导师"角色，节拍11为"复活"场景

### 测试2: 起承转合验证
- **输入**: 文艺情感剧本，选择 `kishotenketsu` 模板
- **验证**: 检查是否生成4个节拍，节拍3为🔴特级关键帧
- **预期**: 节拍1、2情感铺垫充分，节拍4有余韵

### 测试3: 多巴胺闭环验证
- **输入**: 2分钟短剧剧本，选择 `micro_drama_loop` 模板
- **验证**: 检查是否生成4-6个节拍，每45秒有🟠或🔴关键帧
- **预期**: 开头3秒有强烈钩子，结尾有CTA或悬念

### 测试4: 向后兼容验证
- **输入**: 现有三幕式项目
- **验证**: 检查自动迁移为 `classic_three_act`
- **预期**: 现有节拍拆解不变，配置字段更新

## 错误处理

### 结构不匹配错误
- **症状**: 剧本内容与选择的结构严重不匹配
- **处理**: 建议更合适的结构，或切换到兼容模式
- **恢复**: 用户确认后应用新结构或保持兼容

### 验证失败错误
- **症状**: 节拍分布或权重计算不符合结构要求
- **处理**: 提供具体修正建议
- **恢复**: 自动修正或用户手动调整

### 性能问题
- **症状**: 新结构分析时间超过2秒/节拍
- **处理**: 降级到简化分析模式
- **恢复**: 输出简化结果并标记性能警告

```

  ↓

  ↓
[情感细腻、文艺风格] → 推荐 kishotenketsu  
  ↓
[短视频、社交媒体] → 推荐 micro_drama_loop
  ↓
[传统叙事、无特殊要求] → 推荐 classic_three_act
  ↓
用户确认选择
  ↓
应用结构模板
```
