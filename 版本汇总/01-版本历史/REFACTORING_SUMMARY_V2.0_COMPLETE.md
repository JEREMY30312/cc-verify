## 🏗️ 核心成果

### 1. 认知驱动的镜头设计体系（模块 1）

#### NDI（叙事密度指数）
- 计算公式：NDI_score = (动词密度 × 0.4) + (情绪转折点 × 0.35) + (角色位移 × 0.25)
- NDI分级与镜头策略映射：
  - 高 NDI（≥2.5）→ 多镜头原子拆分
  - 中 NDI（1.5-2.4）→ 标准镜头组合
  - 低 NDI（<1.5）→ 长镜头或氛围渲染

#### 镜头合并协议
- **深度构图优先**：前景与背景互动优先通过"四层环境结构"在一个镜头内解决
- **焦点平移判定**：多人交互且无第三方干扰时使用"焦点切换"而非切镜

#### 原子化叙事原则
- 每个原子单元必须锚定"一个核心叙事意图"与"一个视觉重心"
- 镜头生成规则：
  - 单叙事意图 = 单镜头
  - 多叙事意图 = 多镜头
  - 单视觉重心 = 无需拆分
  - 视觉重心转移 = 需要拆分

#### 镜头类型标签
- **Pulse（强动作）**：高 NDI，多动作单元
- **Flow（长镜头）**：低 NDI，氛围渲染
- **Encounter（交互）**：中 NDI，角色互动

#### 镜头动机
- 威慑、悬疑、情感受创、冲突升级、氛围渲染、信息传递
- 无效切镜判定标准：
  - ❌ 无明确动机的切镜
  - ❌ 重复相同叙事意图的镜头
  - ❌ 不改变视觉重心的重复景别
  - ❌ 仅为了"丰富画面"的装饰性切镜

### 3. 动态展示体系（模块 2）

#### 三级展示规则
| 关键帧等级 | 展示规则 | 说明 |
|-----------|----------|------|
| 🔴 特级 (≥3.0) | **强制展示** | 所有红色关键帧必须作为视觉锚点，不允许遗漏 |
| 🟠 高级 (2.0-2.9) | **强制展示** | 所有橙色关键帧必须作为视觉锚点，不允许遗漏 |
| 🟡 中级 (1.0-1.9) | **弹性展示** | 根据节拍节奏筛选，关键动机转折则选入展示名单 |
| 🔵 普通 (0.5-0.9) | **剔除展示** | 仅存在于 scene-breakdown.json 水下全量数据中 |
| ⚪ 次要 (<0.5) | **剔除展示** | 仅存在于 scene-breakdown.json 水下全量数据中 |

#### 🟡 中级关键帧筛选规则

**选入条件（满足其一即可）**：
1. **节奏支撑**: 两处高级关键帧之间跨度过大（超过 5 个节拍），需要中级关键帧作为"节奏支柱"
2. **动机转折**: 该关键帧包含关键的"动机转折"（如：威慑→悬疑、冲突升级→情感受创）
3. **空间连贯**: 该关键帧是轴线跨越的关键接力点

**剔除条件**：
1. 高级关键帧间距合理（≤5 个节拍）
2. 无明显动机转折
3. 不是轴线跨越的关键接力点
4. 已被邻近的高级关键帧覆盖叙事功能

#### 展示规模控制
| 场景特征 | 推荐展示规模 | 说明 |
|----------|-------------|------|
| 高密度动作场景 | 7-9 格 | 多 🔴/🟠，节奏紧凑，充分拆分 |
| 中密度冲突场景 | 5-7 格 | 🔴/🟠 为主，少量 🟡 择优 |
| 低密度氛围场景 | 3-5 格 | 🟡 为主，少量 🔴/🟠 突出高光 |

### 4. 空间守恒约束（模块 1）

#### 轴线法律（Axis Law）
- 所有镜头生成必须核对 scene_breakdown.json 中的 axis_side 字段
- 确保在交互场景中摄影机不发生越轴穿帮
- 轴线侧定义：左侧/右侧/中央
- 跨越轴线必须通过合理调度（如：角色移动、时间跳跃、场景切换）

### 5. 完善的质量检查体系

#### 两层质量检查覆盖所有内容
| 层级 | 执行者 | 检查内容 | 时机 | 失败处理 |
|------|--------|----------|------|----------|
| L1 技术自检 | SKILL 内部 | 格式正确性、数量约束、结构完整性 | 生成过程中 | SKILL 自动修正，无法修正则报错 |
| L2 专业审核 | director | 叙事结构、镜头语言、专业标准、覆盖度、一致性、继承性 | 生成完成后 | FAIL → 返回步骤1修改 |

#### 新增审核项（模块 1 + 模块 2）

**模块 1 新增**:
- 认知审核：审核切镜是否具备明确的"叙事动机"，严禁无效切镜

**模块 2 新增**:
- 关键帧覆盖度：确保所有 🔴 和 🟠 关键帧都在 Beatboard 中展示
- 🟡 中级关键帧合理性：验证选入原因是否合理
- 展示规模合理性：确保展示数量忠实反映剧本节奏
- 冰山完整性：确认展示层和全量层（scene-breakdown.json）的完整性

### 6. 动态布局系统（模块 2）

#### 布局类型映射
| 展示数量 | 推荐布局 | 说明 | Markdown 排版 |

## 📊 数据流变化

### 旧流程（V1.0）
```
剧本 → beat-breakdown-{集数}.md → beat-board-prompt-{集数}.md → sequence-board-prompt-{集数}.md → motion-prompt-{集数}.md
```

### 新流程（V2.0）
```
剧本 → [beat-breakdown-{集数}.md + scene-breakdown-{集数}.json]
       ↓ (全量数据层)
beat-board-prompt-{集数}.md (动态 3-9 格，基于 🔴/🟠/🟡 权重筛选)
       ↓ (读取 scene-breakdown-{集数}.json)
sequence-board-prompt-{集数}.md
       ↓ (读取 scene-breakdown-{集数}.json)
motion-prompt-{集数}.md
```

**关键变化**:
1. 节拍拆解阶段同时生成 MD 和 JSON
2. Beatboard 从固定 4 格改为动态 3-9 格
3. 展示策略基于关键帧权重（🔴/🟠/🟡）
4. 所有后续阶段强制读取 scene-breakdown-{集数}.json
5. 全量数据层贯穿整个流程

## 📝 总结

本次重构成功实现了两个核心转变：

1. **模块 1：认知驱动**：从机械切镜转向认知驱动的镜头设计
   - NDI（叙事密度指数）量化叙事复杂度
   - 镜头合并协议避免机械切镜
   - 原子化叙事原则确保每个镜头有明确动机
   - 建立全量数据层（scene-breakdown-{集数}.json）

2. **模块 2：动态展示**：从固定 4 格到基于权重的动态展示
   - 展示规模动态生成（3-9 格）
   - 🔴 特级和 🟠 高级关键帧强制展示
   - 🟡 中级关键帧弹性展示
   - 展示规模忠实反映剧本节奏

**核心价值**:
1. **认知驱动**: 镜头设计基于叙事动机，而非机械切镜
2. **动态展示**: 展示规模基于关键帧权重，而非固定限制
3. **数据层**: 结构化数据支持继承、验证、可视化
4. **审核完善**: 多维度审核确保质量
5. **空间守恒**: 轴线法律确保交互场景的连续性
