## 数据流向验证

### 修改后的完整数据流

```
剧本 → beat-analyzer.md (规则1+2) → beat-breakdown-template.md (新增字段）

  → quality-check.md (规则5: 完整性校验) → 最终输出
```

### 关键数据传递点

1. **类型化权重传递**:
   - beat-analyzer.md: 计算类型化权重
   - beat-breakdown-template.md: 输出最终权重和修正原因

2. **感官锚点传递**:
   - beat-analyzer.md: 识别并标记感官锚点
   - beat-breakdown-template.md: 输出`[感官:类型/强度]`标签

3. **智能选帧传递**:
   - keyframe-selector.md: 根据策略标签和复杂度选择关键帧
   - beat-board-template.md: 引用选帧依据

4. **场景骨肉融合传递**:
   - beat-breakdown-template.md: 输出策略标签和关键信息点
   - beat-board-template.md: 应用Prompt组装协议生成画面描述

5. **完整性校验传递**:
   - quality-check.md: 执行台词落地率、动作完整度、视觉连贯性检查
   - 所有阶段: 根据校验结果修复问题
