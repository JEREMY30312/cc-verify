## 规则文档更新

### 1. beatboard-workflow.md

**更新位置**：步骤8.5 "执行V4.1-CRITICAL单帧原则统一检查"

**新增规则**：

#### 禁止连续动作序列

| 错误示例 | 正确示例 |
|---------|---------|
| "包子猪被拉走，重重按在案板" | "包子猪被按在案板中央，表情惊讶" |
| "惊恐→系餐巾→期待敲盘" | "包子猪系好餐巾后，期待敲盘" |
| "暴怒→气劲涌动→包子诞生" | "暗物质包子诞生：悬浮红色闪电" |
| "鲁达冲出→滑铲→接球" | "鲁达滑铲姿势清晰" |
| "镇关西暴怒→周身爆气→包子悬浮" | "暗物质包子诞生：案板中央悬浮" |

#### 禁止时间序列词汇

| 禁用词 | 允许使用 | 说明 |
|--------|---------|------|
| "然后"、"接着"、"随后"、"接下来" | 去除 | 表示时间进程，违反单帧 |
| "后" | 在"后"表示完成时允许 | "系好餐巾后"（完成非时间序列）|

#### 强制单帧画面

**格式**：
```
单帧画面 = [动作主体] + [静态姿态] + [表情/眼神]
```

**示例**：
- ✅ "包子猪被按在案板，表情惊讶"
- ✅ "包子猪系好餐巾后，期待敲盘"
- ✅ "暗物质包子诞生：案板中央悬浮"

#### 识别违规格子

**检测方法**：
- 扫描画面描述中的违禁词
- 检测连续动作（识别多个动词+方向词）
- 检测蒙太奇标识（"→"、"→"、"→"）

#### 应用修复算法

**参考案例**：

案例1：去除连续动作
- 错误："包子猪被拉走，重重按在案板"
- 修复："包子猪被抓住后，按在案板中央"

案例2：拆解蒙太奇
- 错误："惊恐→系餐巾→期待敲盘"
- 修复："包子猪系好餐巾后，期待敲盘"

#### 验证修复后

**检查清单**：
- [ ] 所有格子的画面描述都是单一画面（连续动作数=0）
- [ ] 没有违禁词（"然后"、"接着"、"随后"等）

- [ ] 每个格子都有 **风格标签**

#### 验证失败处理

- 如仍有违规格子 → 立即报错，不保存文件
- 错误信息："九宫格违反V4.1-CRITICAL单帧原则，请修复"

### 2. .claude/common/keyframe-selector.md (新建)

**文件用途**：从节拍拆解表的完整描述中提取戏剧张力最强的单帧画面

**核心内容**：

#### V4.1-CRITICAL单帧原则

```
⚠️ 九宫格核心原则：每格=一个关键帧 [V4.1-CRITICAL]
```

**禁止内容**：
- 连续动作序列
- 时间序列词汇（"然后"、"接着"、"随后"）
- 蒙太奇过程（属于四宫格工作）
- 一格多画面

**强制内容**：
- 聚焦单帧：静态姿态 + 表情 + 眼神
- 格式：[动作主体] + [静态姿态] + [表情/眼神]
- **[关键帧]** ✅标签
- **风格标签**

#### 单帧提取算法

**步骤**：

1. 识别动作序列（检测违禁词和动作动词）
2. 定位戏剧张力最强帧（参考beat-breakdown.md的"戏剧张力最强帧"列）
3. 单帧画面重新表述（去除连续动作，禁用违禁词，聚焦单帧画面）
4. 验证单帧合规性（检查清单）
5. 生成备选帧（JSON.alternative_frames）

**处理示例**：

示例1：节拍3（动势组）
- 输入：鲁达张望→抓猪→按在案板
- 单帧提取："包子猪被按在案板，表情惊讶"
- 备选帧：起势（鲁达张望+1.2），动作开始（抓猪+1.35）

示例2：节拍4（蒙太奇组）
- 输入：惊恐→听到给它吃→系餐巾→期待敲盘
- 单帧提取："包子猪期待敲盘"
- 备选帧：起势（惊恐+1.3），转折（听到+1.5），动作（系餐巾+1.7）

#### 质量保证

**单帧合规率目标：100%**

**违规检测**：
- 连续动作动作：0%
- 时间序列词：0%
- 方向性动作：0%
- 蒙太奇标识：0%
- 格式合规（**[关键帧]**标签）：100%

## 数据流向规范化（V4.1）

### 正确数据流

```
scene-breakdown.json shots (完整原文)
    ↓
节拍拆解提取单帧片段
    ↓
beat-breakdown.md "戏剧张力最强帧"列
    ↓
beat-board-full-list.json scene_description (单帧)
    ↓
beat-board-prompt-*.md 格子画面描述 (单帧)
    ↓
四宫格展开：使用alternative_frames + 原文scene_description
```

### 关键字段对比

| 文件 | scene_description | 内容 | 用途 |
|------|-----------------|------|------|
| scene-breakdown.json | S02 | 完整原文："鲁达睁眼...镇关西切臊子...挑衅回应" | 原始数据源 |
| beat-board-full-list.json | F02 | 单帧片段："鲁达睁开挑剔双眼傲慢表达" | 九宫格显示 |
| beat-board-prompt.md | 格子2 | 单帧描述（与JSON一致）| 九宫格显示 |

### alternative_frames用途

**JSON.alternative_frames**：
```json
{
  "alternative_frames": [
    {
      "frame_type": "起势",
      "role": "中景",
      "action": "鲁达张望，目光落在包子猪身上",
      "drama_tension_score": 1.2,
      "reason": "戏剧张力较弱"
    },
    {
      "frame_type": "动作开始（抓猪）",
      "role": "中景",
      "action": "鲁达伸出手抓住包子猪",
      "drama_tension_score": 1.35,
      "reason": "戏剧张力强但不如按在案板帧"
    }
  ]
}
```

**四宫格使用**：
- 读取alternative_frames
- 读取原文scene_description
- 展开为四宫格：起势→动作→结果
