# 创意保留策略设计方案

## 策略核心思想

### 1. 创意元素分级系统

基于创意价值和对画面描述的重要性，将创意元素分为四个优先级：

#### 高优先级（P1 - 必须保留）
- **定义**：对画面描述至关重要,删除会导致画面显著变样或失去专业性的元素
- **类别**：
  - 电影技术术语：荷兰角、低机位、高角度、仰拍、俯拍、浅景深、深景深
  - 关键镜头参数：24mm、85mm、广角、特写、中景、极远景
  - 高级光影效果：轮廓光、侧光、丁达尔效应、体积光、眼神光
  - 关键情感细节：瞳孔收缩、汗珠滑落、发丝光晕、嘴角微表情
- **保留率目标**：95-100%

#### 中高优先级（P2 - 尽量保留）
- **定义**：对画面细节和氛围有重要贡献，删除会降低画面丰富度的元素
- **类别**：
  - 视觉隐喻元素：牢笼般、翅膀、剪影、残影、光斑
  - 环境质感：虚化、散景、色块、质感、纹理
  - 色彩色调：饱和度、色温、色调、对比度
  - 动作细节：呼吸可见、手指描摹、身体前倾
- **保留率目标**：80-90%

#### 中优先级（P3 - 可适度精简）
- **定义**：对画面描述有辅助作用,但可以适当合并或简化表达
- **类别**：
  - 部分修饰词：非常、极其、特别、显得
  - 重复性描述：相似的形容词堆叠
  - 部分环境细节：不关键的环境设定
- **保留率目标**：60-75%

#### 低优先级（P4 - 可大幅精简）
- **定义**：对核心画面描述影响较小,可以大幅压缩或删除
- **类别**：
  - 过渡性词语：然后、接着、接下来
  - 冗余说明：括号内的补充说明
  - 非核心形容词：华丽但不必要的修饰
- **保留率目标**：30-50%

## 智能精简策略

### 1. 三层渐进精简

```
第一层：安全区域识别
├─ 标记P1元素（不可精简）
├─ 标记P2元素（谨慎精简）
└─ 标记P3/P4元素（可精简区域）

第二层：智能压缩
├─ P4元素：删除40-70%
├─ P3元素：合并相似描述，删除25-40%
└─ P2元素：轻微调整，删除5-15%

第三层：质量检查
├─ 创意元素保留率检查
├─ 质量维度评分验证
└─ 电影语言完整性检查
```

### 2. 动态Token分配

根据内容创意密度动态调整精简程度：

| 创意密度 | Token分配 | 精简程度 | 预期精简率 |
|---------|-----------|---------|-----------|
| 高（>15%） | 120% | 轻度精简 | 20-25% |
| 中（10-15%） | 100% | 适度精简 | 30-35% |
| 低（<10%） | 80% | 较大精简 | 40-45% |

### 3. 创意核心保护机制

#### 保护算法
1. **创意元素检测**：识别所有P1和P2元素
2. **安全区划定**：标记包含创意元素的句子为安全区
3. **精简边界设定**：在安全区边界进行精简
4. **完整性验证**：确保创意元素上下文完整

#### 保护示例

```
原文：
头顶荧光灯闪烁，刺眼的光线将两人的脸分割成明暗两半。
身后混凝土柱子形成牢笼般的框架。

创意元素：闪烁、光线、明暗两半、牢笼般、框架
保护策略：
- 闪烁（P2）→ 保留
- 光线（P2）→ 保留
- 明暗两半（P2）→ 保留
- 牢笼般（P2）→ 保留为视觉隐喻
- 框架（P2）→ 保留

简化后：
头顶灯闪烁，光线分割两人的脸为明暗。后方柱子形成牢笼框架。
（删除：刺眼的、混凝土、的）
```

## 多质量维度保护

### 1. 电影质量维度保护阈值

| 维度 | 权重 | 最低保留率 | 警告阈值 |
|------|------|-----------|---------|
| 电影技术 | 1.0 | 85% | <80% |
| 情感深度 | 0.9 | 80% | <75% |
| 视觉丰富度 | 0.8 | 75% | <70% |
| 叙事清晰度 | 0.7 | 70% | <65% |

### 2. 失效保护机制

当任一维度低于警告阈值时：
1. **暂停精简**：停止进一步的精简操作
2. **回滚恢复**：恢复到上一版本
3. **触发警报**：通知需要人工审核
4. **记录问题**：记录到质量日志

## 三种输出模式

### 完整模式（Mode 1 - Full）
- **目标**：创意质量最大化，Token使用100%
- **保留率**：P1 100%，P2 95%，P3 80%
- **适用场景**：高质量需求、关键场景
- **预期精简**：10-15%
- **创意质量**：≤5%下降

### 标准模式（Mode 2 - Standard）
- **目标**：质量与性能平衡，Token使用70%
- **保留率**：P1 100%，P2 85%，P3 60%
- **适用场景**：一般使用场景
- **预期精简**：30-35%
- **创意质量**：≤10%下降

### 快速模式（Mode 3 - Fast）
- **目标**：最佳性能，Token使用50%
- **保留率**：P1 95%，P2 70%，P3 40%
- **适用场景**：快速预览、批量处理
- **预期精简**：50-60%
- **创意质量**：≤20%下降

## 风险控制

### 高风险项
1. **创意元素识别不准确**
   - 缓解：建立专业标注数据集，持续优化算法
   - 回退：保留完整版本快速回滚能力

2. **质量阈值设定不合理**
   - 缓解：小范围试点验证，多轮调整参数
   - 回退：建立人工审核机制，及时发现问题

### 中风险项
1. **Token节省效果不达预期**
   - 缓解：实施三种模式，满足不同需求
   - 回退：建立动态调整机制，持续优化

2. **系统复杂度增加**
   - 缓解：模块化设计，清晰文档
   - 回退：保留简化版本作为备选方案

## 长期优化方向
