# ANINEO 版本汇总

> 所有非核心运行文件的集中管理目录

---

## 📋 目录结构

```
版本汇总/
├── 01-版本历史/          # 版本历史、重构记录、测试报告
├── 02-测试验证/          # 测试数据和测试结果
├── 03-监控系统插件/      # 批处理、监控、警报系统
├── 04-知识库优化插件/    # 知识库管理和优化工具
├── 05-临时文件归档/      # 历史临时文件和备份
├── 06-辅助工具/          # 系统辅助工具
├── 临时池/              # 新文件暂存区
├── README.md            # 本文件
└── 整理完成报告.md      # 整理历史记录
```

---

## ⚡ 新增文件快速指引（三原则）

### 原则1: 根目录仅保留AGENTS.md

**核心系统目录（不变动）**：
- `.claude/` - 核心技能和通用模块
- `.opencode/` - 命令系统
- `agents/` - 子Agent配置
- `configs/` - 配置目录
- `outputs/` - 输出目录
- `script/` - 剧本目录
- `.strategic-snapshots/` - 快照系统
- `.templates/` - 模板系统
- `.sisyphus/` - 草稿系统

**❌ 严格禁止在根目录创建**（除非系统启动脚本）：
- 任何 `.py` 脚本文件
- 任何 `.sh` 脚本文件
- 任何 `.js` 脚本文件
- 任何 `.md` 文档文件（除了 AGENTS.md）
- 任何临时测试文件

---

### 原则2: 新文件必问：这个文件属于01-06哪个分类？

#### 如果知道分类：

| 文件类型 | 放入目录 |
|---------|---------|
| 版本历史、重构记录 | `01-版本历史/` |
| 测试数据、测试结果 | `02-测试验证/` |
| **新插件代码** | `XX-插件名称/`（见原则3） |
| 历史临时文件 | `05-临时文件归档/` |
| 辅助工具 | `06-辅助工具/` |

#### 如果不知道分类：

**立即放入 `版本汇总/临时池/`**
- 不影响工作，先暂存
- 定期整理到01-06对应分类
- 或直接删除（如果无价值）

```
临时池子目录（按类型暂存）：
├── 暂存脚本/     # .py, .sh, .js 等脚本
├── 暂存文档/     # .md, .txt 等文档
├── 暂存数据/     # .json, .yaml 等数据
└── 不确定文件/   # 其他不确定的文件
```

---

### 原则3: 新增插件参照03/04现有结构

#### 插件目录结构模板：

```
版本汇总/XX-插件名称/
├── README.md                    # [必须] 插件说明文档
├── 核心功能/                    # [必须] 核心代码
│   ├── main_script.py
│   └── helper.py
├── 配置文件/                    # [推荐] 配置文件
│   ├── config.json
│   └── settings.yaml
├── 文档/                        # [推荐] 使用文档
│   ├── USER_GUIDE.md
│   └── CHANGELOG.md
└── logs/                        # [可选] 日志目录
```

#### 插件README必须包含：

```markdown
# XX插件

## 概述
[插件功能一句话描述]

## 核心功能
1. [功能1]
2. [功能2]

## 使用方法
```bash
python 版本汇总/XX-插件名称/核心功能/main_script.py
```

## 目录结构
[插件目录结构说明]

---
版本: V1.0
更新时间: YYYY-MM-DD
```

## 🎯 典型场景示例

### 场景1: 开发测试新功能

**问题**: 开发新的批处理功能，产生了很多测试文件

**解决**:
```
开发中 → 版本汇总/临时池/暂存脚本/
测试数据 → 版本汇总/临时池/暂存数据/
完成开发 → 创建 05-批处理优化插件/
```

---

### 场景2: 临时测试脚本

**问题**: 需要快速测试某个想法，写了test_demo.py

**解决**:
```
立即放入 → 版本汇总/临时池/暂存脚本/test_demo.py
测试完成 → 整理到对应分类 或 删除
```

---

### 场景3: 新增完整插件

**问题**: 开发完成一个新功能插件

**解决**:
```
1. 创建插件目录: 版本汇总/05-新功能插件/
2. 参照03/04插件结构创建子目录
3. 创建README.md（必须）
4. 从临时池移动文件到对应目录
```
