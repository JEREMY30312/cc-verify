# 方案A - 三原则快速参考卡

> 随时可用的快速查找表

---

## 🎯 三原则（5秒记住）

```
1. 根目录只放核心文件，其他全进版本汇总
2. 不确定放哪里，就先放临时池
3. 新插件参照03/04，必须有README
```

---

## 📂 文件放置指南

| 文件类型 | 放哪儿 |
|---------|-------|
| **系统核心** | .claude/, agents/, .opencode/ 等 |
| **版本历史** | 01-版本历史/ |
| **测试相关** | 02-测试验证/ |
| **新插件** | XX-插件名称/ |
| **历史备份** | 05-临时文件归档/ |
| **辅助工具** | 06-辅助工具/ |
| **🛑 不知道？** | **临时池！** |

---

## 🏗️ 新插件结构模板

```
XX-插件名称/
├── README.md          # 必须
├── 核心功能/          # 必须
├── 配置文件/          # 推荐
├── 文档/              # 推荐
└── logs/              # 可选
```

---

## 🔍 快速查找

```
找什么？        去哪？
━━━━━━━━━━━━━━━━━━
测试结果    → 02-测试验证/

```

## ⚡ 常用命令

```bash
# 检查根目录
./check_root.sh

# 查看临时池
ls 版本汇总/临时池/

# 查看插件示例
cat 版本汇总/03-监控系统插件/README.md
```

---

## ⚠️ 根目录禁止

```
❌ 任何 .py 脚本
❌ 任何 .sh 脚本
❌ 任何 .js 脚本
❌ 任何 .md 文档（除了 AGENTS.md）
❌ 任何临时测试文件
```
