# 智能文件整理器 - 快速参考卡

> **一句话解决文件混乱问题**：运行整理器，忘记文件管理

---

## 🚀 三种启动方式

### 1️⃣ 实时监控（推荐日常使用）

```bash
./start_organizer.sh
```

✅ 持续运行，新文件自动整理
✅ 无需手动操作
✅ 按 Ctrl+C 停止

---

### 2️⃣ 单次扫描（一次性整理）

```bash
./start_organizer.sh --once
```

✅ 扫描当前所有文件
✅ 执行一次后退出
✅ 不需要安装 watchdog

---

### 3️⃣ 试运行（测试用）

```bash
./start_organizer.sh --dry-run
```

✅ 不实际移动文件
✅ 仅显示将要执行的操作
✅ 用于测试规则

## ✨ 核心优势

| 特性 | 优势 |
|------|------|
| **预防性管理** | 从源头防止混乱，而不是事后补救 |
| **零手动操作** | 创建文件时自动整理，无需你关心放哪里 |
| **智能分类** | 基于文件类型、名称、用途自动判断 |
| **轻量级** | 一键启动，零配置，低资源占用 |
| **灵活扩展** | 可自定义分类规则，适应不同需求 |

## 🎯 与三原则法的关系

### 传统三原则（需要手动记忆）

| 原则 | 内容 | 执行方式 |
|------|------|----------|
| 原则1 | 根目录仅保留AGENTS.md | ❌ 手动记忆 + 手动执行 |
| 原则2 | 新文件必问属于01-06哪个分类 | ❌ 手动思考 + 手动移动 |
| 原则3 | 新增插件参照03/04现有结构 | ❌ 手动创建 + 手动组织 |

### 智能整理器（自动执行）

| 原则 | 内容 | 执行方式 |
|------|------|----------|
| 原则1 | 根目录仅保留核心文件 | ✅ 自动识别 + 自动保留 |
| 原则2 | 新文件必问属于01-06哪个分类 | ✅ 智能分析 + 自动分类 |
| 原则3 | 新增插件参照03/04现有结构 | ✅ 自动创建 + 自动组织 |

**关键区别**：
- ❌ 传统方式：需要记忆规则 + 手动思考 + 手动执行
- ✅ 智能整理器：零记忆 + 零思考 + 零操作

## 🔥 典型使用场景

### 场景1：日常开发

```bash
# 终端1：启动整理器（持续运行）
./start_organizer.sh

# 终端2：正常开发
touch test_script.py      # 自动 → 临时池/暂存脚本/
echo "report" > report.md # 自动 → 临时池/暂存文档/

```

```bash
# 先试运行，查看将会执行的操作
./start_organizer.sh --dry-run

# 确认无误后，执行实际整理
./start_organizer.sh --once
```

```bash

# 查看实际移动的文件
ls 版本汇总/临时池/不确定文件/
```

## 🛠️ 故障排查

### 问题：watchdog 库未安装

**解决**：
```bash
pip3 install watchdog
```

或使用单次扫描模式（不需要 watchdog）：
```bash
./start_organizer.sh --once
```

### 问题：文件未移动

可能原因：
1. 文件名以 `.` 开头（隐藏文件，不处理）
2. 文件在 `ALLOWED_ROOT_FILES` 中（核心系统文件）
3. 文件不在根目录（只监控根目录）

**调试**：
```bash
./start_organizer.sh --dry-run
```
