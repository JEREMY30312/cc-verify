## 🎯 三原则详解

### 原则1: 根目录仅保留AGENTS.md

```
❌ 根目录禁止创建：
- 任何 .py 脚本文件
- 任何 .sh 脚本文件
- 任何 .js 脚本文件
- 任何 .md 文档文件（除了 AGENTS.md）

✅ 根目录保留：
- AGENTS.md (核心配置)
- .agent-state.json (状态文件)

```

## ⚠️ 长期风险控制

### 可能遇到的问题

1. **开发者忘记规则**
   - 解决: 定期运行 `./check_root.sh`
   - 避免: 将check_root.sh加入开发工作流

2. **临时池堆积**
   - 解决: 定期清理（每周/每月）
   - 避免: 明确临时池只是暂存，不是永久存放

3. **插件结构不规范**
   - 解决: 参照03/04现有插件结构
   - 避免: 创建插件时先看03/04的README

### 隐藏的坑及防范

| 隐藏坑 | 风险 | 防范措施 |
|--------|------|----------|
| 规则被遗忘 | 文件再次混乱 | 定期运行检查脚本 |
| 临时池变成垃圾场 | 失去整理意义 | 定期清理机制 |
| 插件命名混乱 | 难以管理 | 建议使用 `XX-功能名称` 格式 |
| 核心文件误移 | 系统故障 | AGENTS.md 和核心目录保持不动 |
| README过时 | 规则失效 | 简单原则不需要频繁更新 |
